## AIAEST叙事框架

### 框架定义

| 阶段 | 定义 | 情感目标 | 宣传片章节 |
|------|------|----------|------------|
| **A**ttention | 注意 - 开场吸引 | 好奇/兴趣 | 序章 |
| **I**nterest | 兴趣 - 引发关注 | 投入/好奇 | 第一章 |
| **A**ction | 行动 - 展示作为 | 期待/认同 | 第二章 |
| **E**motion | 情感 - 情感共鸣 | 震撼/感动 | 第三章 |
| **S**atisfaction | 满足 - 价值确认 | 信任/行动 | 终章 |

### 三幕式结构

```
┌─────────────────────────────────────────────────────┐
│                    第一幕：建置 (25-30%)              │
│  序章(10s) + 第一章(60s)                            │
│  • 建立主角（企业/品牌/创始人）                       │
│  • 展示现状或面临的问题                             │
│  • 创造情感连接的第一个钩子                          │
└─────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────┐
│                  第二幕：对抗 (50-60%)                │
│  第二章(60s)                                        │
│  • 详细展示企业的应对策略                           │
│  • 通过故事、数据、案例证明能力                      │
│  • 建立差异化认知                                   │
└─────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────┐
│                   第三幕：解决 (20-25%)              │
│  第三章(60s) + 终章(10s)                          │
│  • 展示成功结果和愿景                              │
│  • 情感高潮的打造                                  │
│  • 明确的行动呼吁                                  │
└─────────────────────────────────────────────────────┘
```

### 坎瓦尔情感曲线

```
情感强度
    ↑
    │    ╭───╮              ╭───╮
    │    │峰值│    ╭───╮    │峰值│
    │    │ E  │    │ E  │    │ E  │
    │    ╰────╯────╰────╯────╰────╯
    └──────────────────────────────→ 时间
       A     I     A     E     S
      (注意)(兴趣)(行动)(情感)(满足)
```

## 自适应询问逻辑

### 问题调整机制

**根据回答质量动态调整：**

```
用户回答质量评估：
├── 模糊/简短 → 触发追问引导
├── 具体/丰富 → 触发深挖追问
├── 跳跃/发散 → 触发聚焦追问
└── 沉默/跳过 → 记录待补充，继续下一问
```

**追问深度层级：**

| 层级 | 触发条件 | 追问策略 |
|------|----------|----------|
| L1-事实层 | 回答笼统 | "能具体说说吗？" |
| L2-原因层 | 回答表面 | "为什么这么想？" |
| L3-意义层 | 回答浅显 | "这意味着什么？" |
| L4-例外层 | 回答确定 | "有没有例外情况？" |
| L5-假设层 | 回答完成 | "如果没有...会怎样？" |

### 核心问题模板

#### Wave 1：企业介绍 + 产品服务

**企业介绍（A-I）**

```
【开场钩子】
Q: "如果用一句话向一个完全不了解你们行业的人介绍公司，你会怎么说？"

【竞争优势深挖】
Q: "您认为，在客户眼中，贵公司与其他同行最大的不同是什么？"

【追问】
追问1: "什么让你认为这是'最大的不同'？"
追问2: "客户是否也这么认为？有证据吗？"

【竞争定位】
Q: "在行业中，贵公司处于什么位置？有没有具体的数据可以说明？"
```

**产品服务（I-A）**

```
【痛点挖掘】
Q: "客户在遇到什么问题时会想到你们？"

【追问】
追问1: "这个问题如果不解决，客户会有什么后果？"
追问2: "客户通常用什么词描述这个问题？"

【差异化】
Q: "你们的产品或服务，有什么让客户觉得'换不了'的因素吗？"

【技术壁垒】
Q: "在技术上，有什么是竞争对手做不到、只有你们能做的？"
```

#### Wave 2：团队实力 + 发展历程

**团队实力（E-情感共鸣）**

```
【创始人故事】
Q: "创始人为什么选择做这件事？最初的动机是什么？"

【STAR深挖】
追问1-S: "当时是什么情况促使这个决定？"
追问2-T: "面临的最大挑战是什么？"
追问3-A: "具体采取了什么行动？"
追问4-R: "结果如何？有数字可以证明吗？"

【团队精神】
Q: "如果用三个词来形容公司的团队氛围，会是哪三个？"

【追问】
追问: "有没有一个具体的场景或故事能体现这种氛围？"
```

**发展历程（E-S）**

```
【高光时刻】
Q: "回顾企业发展历程，哪个时刻最让你感到骄傲？"

【STAR深挖】
S: "当时是什么情况？"
T: "面临什么困难？"
A: "怎么克服的？"
R: "带来了什么改变？"

【至暗时刻】
Q: "有没有遇到过差点过不去的坎？最后怎么过来的？"

【Pre-mortem】
Q: "如果三年后这个企业宣传片失败了，您觉得最可能的原因是什么？"
```

#### Wave 3：未来愿景 + 合作案例

**未来愿景（A-I）**

```
【愿景描绘】
Q: "三到五年后，您希望公司成为什么样子？"

【追问】
追问1: "为什么这个目标对您重要？"
追问2: "达到这个目标的标志是什么？"

【社会价值】
Q: "除了商业成功，公司还想给世界带来什么？"

【追问】
追问: "这个社会价值和商业模式是如何结合的？"
```

**合作案例（S-信任）**

```
【成功故事】
Q: "有没有一个客户案例，最能证明你们带来的价值？"

【STAR法则】
S: "当时客户面临什么情况？"
T: "想要达成什么结果？"
A: "你们具体做了什么？"
R: "最终效果如何？有数据吗？"

【信任背书】
Q: "行业內有没有什么权威认证或奖项，能证明你们的实力？"
```

#### Wave 4：受众分析 + 竞争分析

**受众分析（A-E）**

```
【受众画像】
Q: "这部宣传片主要是给谁看的？他们的年龄、职位、关注点是什么？"

【深层理解】
追问1: "他们最关心什么？最担心什么？"
追问2: "他们通常从什么渠道了解企业信息？"

【情感设计】
Q: "观众看完宣传片后，您希望他们心里想的是什么？"

【行动号召】
Q: "您希望观众看完后做什么？"
```

**竞争分析（E-差异化）**

```
【竞争定位】
Q: "如果只能选一个最强的竞争对手，是谁？"

【差异深挖】
Q: "客户选择你们而不是他们的理由是什么？"

【壁垒验证】
Q: "有什么护城河是对手短期难以复制的？"

【战略透镜】
追问: "竞争对手最可能攻击你们哪一点？"
追问: "你们有什么潜在优势还没被认识到？"
```

#### Wave 5：风险识别 + 叙事设计

**风险识别（S）**

```
【敏感话题】
Q: "有什么话题或信息，最好不要在宣传片中提及？"

【合规要求】
Q: "在品牌规范或法律合规方面，有什么必须遵守的要求？"

【担忧预判】
Q: "您最担心宣传片出来后，哪方面的反馈？"
```

**叙事设计（整合）**

```
【企业精神】
Q: "如果用一句话概括企业的灵魂或精神，会是什么？"

【核心故事】
Q: "有没有一个故事，最能代表这家公司？"

【象征意象】
Q: "如果用一个画面或意象代表企业，会是什么？"

【叙事风格】
Q: "您喜欢什么风格的宣传片？有没有参考案例？"

【心理透镜】
追问: "您说的是'我们想成为的'，还是'我们本来就是的'？"
追问: "这个故事背后，真正想传递的信息是什么？"
```

## 质量评估体系

### 完整度评估

```
基础维度完整度 = (完成标签数 / 最低要求标签数) × 100%
增强维度完整度 = (完成标签数 / 最低要求标签数) × 100%
总体完整度 = (基础×0.6 + 增强×0.4) × 100%

最低要求标签：
- 企业介绍: 企业名称,行业,规模,核心优势 (4)
- 产品服务: 产品线,核心产品,解决痛点,差异化 (4)
- 团队实力: 创始人,核心团队,企业文化 (3)
- 发展历程: 里程碑≥2,关键转型 (3)
- 未来愿景: 发展目标,战略方向 (2)
- 合作案例: 标杆客户≥1,合作伙伴 (2)
- 受众分析: 主要受众,关注点,情感诉求 (3)
- 竞争分析: 竞争对手,核心差异,竞争壁垒 (3)
- 风险识别: 挑战,敏感话题 (2)
- 叙事设计: 企业精神,核心故事,象征意象 (3)
```

### 叙事完整性检查

```
[ ] 是否有一条清晰的主线贯穿全片？
[ ] 是否在开头制造了好奇或悬念？
[ ] 是否有至少一个情感共鸣点？
[ ] 是否在高潮部分展示了独特的差异化？
[ ] 是否在结尾有明确的行动号召？
[ ] 是否避免了信息堆砌（核心信息≤3个）？
[ ] 情感曲线是否合理（渐进而非突变）？
```

### 差异化验证

```
[ ] 是否明确说明了"为什么选你而不是竞争对手"？
[ ] 差异化是否有具体的证据或数据支撑？
[ ] 差异化是否可感知（而非自说自话）？
[ ] 是否有竞争对手做不到、只有你们能做的事？
```

## 状态机 v3.0

```
[INIT] → [WAVE_1] → [WAVE_2] → [WAVE_3] → [WAVE_4] → [WAVE_5] → [ANALYZING] → [NARRATING] → [GENERATING] → [REFINING] → [COMPLETE]
              ↓           ↓           ↓           ↓           ↓
              └───────────┴───────────┴───────────┴───────────┘
                         (可随时补充信息，返回上一波浪)

[REFINING]: 根据质量评估结果，补充缺失信息或优化现有内容
```

### 状态定义

| 状态 | 说明 | 进入条件 |
|------|------|----------|
| INIT | 初始化项目 | 首次启动 |
| WAVE_1-5 | 波浪追问 | 依次进入 |
| ANALYZING | 整合分析 | 完成WAVE_5或用户要求 |
| NARRATING | 叙事提炼 | ANALYZING后 |
| GENERATING | 生成文案 | 用户确认或完整度>75% |
| REFINING | 质量打磨 | GENERATING后自检或用户反馈 |
| COMPLETE | 完成 | 用户确认或无进一步修改 |
