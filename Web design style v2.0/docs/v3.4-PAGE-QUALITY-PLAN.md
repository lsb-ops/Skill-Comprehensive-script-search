# v3.4 · 网页生成效果深度迭代

> **核心目标**: 从"能跑"升级到"专业级",主题适配 + 视觉丰富 + 真实美学
> **范围**: 4 个新文件 + 6 个改动 + 1 个测试
> **基线**: v3.3 74/74 tests, 9 layouts, 6 schemes, 2 modes
> **方向**: 不再围绕抖音,聚焦 reveal.js 网页生成效果

---

## Context — 为什么这次迭代

v3.3 把"声明 vs 实际"补全了(74/74 tests),但**用户站在浏览器前看 index.html 时还是不满意**。
具体问题:

| 现状 | 视觉感受 |
|------|----------|
| 9 种 layout 随机循环 | 同一套 layout 给"AI 教程"和"咖啡真相"用,完全不专业 |
| `layout-waterfall` 只有 `.title`/`.body` CSS | 名义上是瀑布流,实际是普通居中,空架子 |
| `layout-compare` 硬编码 `#FFE5E5`/`#E5F7E5` | 赛博朋克主题下绿红对比违和 |
| 大部分页纯色背景 | 缺装饰元素,缺视觉层次 |
| 所有 heading 800-900 weight | 缺中粗/常规层级,平面化 |
| 21 条 anti-slop 全是结构/颜色 | 缺视觉层级/装饰/字距规则 |

**最致命的 1 个**: 不同主题用同一套 layout 序列,这是"工具感"而非"作品感"。

## 4 个新发现 (深度分析)

### 1. 主题-布局映射缺失

```python
# 当前: apply_variance 9 种循环,不分类
"AI 教程" → [卡片, 大字报, 数字, 对比, 图标, 时间线, 故事线, 问答, 卡片]
"咖啡真相" → [卡片, 大字报, 数字, 对比, 图标, 时间线, 故事线, 问答, 卡片]
```

**应有**:
```python
"AI 教程" → [问答, 时间线, 对比, 数字, 卡片]      # 解释/步骤型
"咖啡真相" → [大字报, 数字, 故事线, 卡片, 图标]   # 揭秘/启发型
"产品介绍" → [大字报, 对比, 数字, 卡片, 问答]      # 价值/差异型
"故事叙述" → [故事线, 大字报, 卡片, 卡片, 图标]   # 情感/节奏型
```

### 2. `waterfall.css` 死架子

```css
/* 当前: 只有 .title/.body,完全没有 waterfall 结构 */
.layout-waterfall { align-items: stretch; text-align: left; gap: var(--space-sm); }
.layout-waterfall .title { font-size: var(--font-size-h1); margin-bottom: var(--space-sm); }
.layout-waterfall .body { text-align: left; max-width: 90%; }
```

**应有**: CSS columns masonry,3 列自动排。

### 3. `compare.css` 硬编码红绿

```css
/* 当前: 不分主题硬编码,赛博朋克/莫兰迪都红绿 */
.layout-compare .compare-card.before { background: #FFE5E5; color: #C92A2A; }
.layout-compare .compare-card.after { background: #E5F7E5; color: #2A9D2A; }
```

**应有**: 用 CSS variable `--compare-before`/`--compare-after`,从 color_scheme 注入。

### 4. 缺装饰元素

当前 9 种 layout 都是"文字 + 背景",无:
- 几何装饰 (圆点/线条/方块)
- 渐变光晕
- 数字徽章
- 装饰边框

**应有**: 每种 layout 至少 1 个装饰元素类(如 `.deco-dots`/`.deco-line`/`.deco-glow`)。

## 实施阶段

### Phase 1: 主题分类 + 布局选择 (HIGH)

**新建 `scripts/topic_layout.py`**:

```python
# 5 种内容主题 → 推荐 layout 序列
TOPIC_LAYOUT_THEMES = {
    "tutorial": {  # 教程/AI/技术
        "keywords": ["ai", "教程", "学习", "代码", "工具", "方法", "步骤", "如何", "怎么"],
        "preferred_layouts": ["qa", "timeline", "compare", "numbers", "card"],
        "motion": "fragment-fade-up",
        "rationale": "解释步骤需要时间线/对比/问答",
    },
    "listicle": {  # 列表/真相/排行
        "keywords": ["真相", "个", "招", "种", "盘点", "排行", "top", "秘密"],
        "preferred_layouts": ["numbers", "icon", "card", "poster", "card"],
        "motion": "fragment-zoom-in",
        "rationale": "数字+图标+卡片是列表黄金组合",
    },
    "story": {  # 故事/情感
        "keywords": ["故事", "经历", "我", "我们", "人生", "感受", "情感"],
        "preferred_layouts": ["storyline", "poster", "card", "icon", "waterfall"],
        "motion": "fragment-slide-left",
        "rationale": "故事需要叙事节奏+情感大字+碎片",
    },
    "product": {  # 产品/工具
        "keywords": ["产品", "工具", "app", "软件", "平台", "推荐"],
        "preferred_layouts": ["poster", "compare", "numbers", "card", "qa"],
        "motion": "fragment-bounce",
        "rationale": "产品需要大字+对比+量化价值",
    },
    "general": {  # 兜底
        "keywords": [],
        "preferred_layouts": ["card", "icon", "numbers", "timeline", "qa"],
        "motion": "fragment-fade-up",
    },
}

def classify_topic(topic: str, points: list) -> str:
    """根据 topic + points 关键词分类到 5 种主题之一"""
    text = topic + " " + " ".join(_get_title(p) for p in points)
    text_lower = text.lower()
    scores = {}
    for theme, cfg in TOPIC_LAYOUT_THEMES.items():
        if theme == "general":
            continue
        scores[theme] = sum(1 for kw in cfg["keywords"] if kw in text_lower)
    # 最高分胜出,平局回 general
    if not scores or max(scores.values()) == 0:
        return "general"
    return max(scores, key=scores.get)


def select_layouts(topic: str, points: list, variance: int, mode: str = "portrait") -> list:
    """
    根据 topic 主题 + variance 选 layout 序列。
    variance=1: 全部用 preferred[0]
    variance=5: 前 5 个 preferred
    variance=10: 9 个全部打乱但保留 preferred 优先
    """
    theme = classify_topic(topic, points)
    preferred = TOPIC_LAYOUT_THEMES[theme]["preferred_layouts"]
    n = max(1, min(len(preferred), variance))  # 1-9 之间
    layouts = preferred[:n]
    # variance=10 时: 补齐 9 种(从其他主题借,但 preferred 仍在前)
    if variance >= 9:
        all_others = [l for t in TOPIC_LAYOUT_THEMES.values() for l in t["preferred_layouts"]]
        # 去重 + 保序
        seen = set()
        rest = []
        for l in all_others:
            if l not in layouts and l not in seen:
                rest.append(l)
                seen.add(l)
        layouts = layouts + rest[:9 - len(layouts)]
    return layouts
```

**修改 `apply_variance`**: 不再硬循环 9 种,接受 `topic` 参数,调 `select_layouts`。

### Phase 4: 视觉层级 anti-slop 规则 (MEDIUM)

**扩展 anti-slop 从 21 → 30 条**,新增 9 条视觉规则:

| ID | 规则 | 严重度 |
|----|------|--------|
| AS-22 | 单页只 1 个 `.title`/`.body` 元素 → fail | medium |
| AS-23 | 单页 visual 元素数 = 0 → fail (太单调) | medium |
| AS-24 | 同 layout 连续 ≥ 3 页 → warn (缺多样性) | low |
| AS-25 | 标题与正文字号差 < 4px → fail (缺层级) | high |
| AS-26 | 全 9 页同 color-scheme 但 accent 全 0 → fail (accent 没生效) | high |
| AS-27 | `.body` 长度 > 200 字未断行 → warn | low |
| AS-28 | CTA 页无 `.emoji` 或 `.icon` → warn | low |
| AS-29 | 第一页 cover 无 `.subtitle` → fail | medium |
| AS-30 | 末页 CTA 与第一页 cover bg 不同色 → warn | low |

### Phase 5: typography 细化 (LOW)

- `--font-weight-body: 400` (常规) / `--font-weight-emphasis: 500` (中等) / `--font-weight-heading: 700` (heading) / `--font-weight-display: 900` (display)
- 中文字距: `letter-spacing: 0.02em` for body, `-0.01em` for display
- 行高: `line-height: 1.6` for body, `1.2` for display
