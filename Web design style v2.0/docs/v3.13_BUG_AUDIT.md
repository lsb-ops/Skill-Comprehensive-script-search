## 🔴 P0 必修 bug (4 个)

### BUG-01 · harmony-check.js:61-62 · 字段不一致
**位置**: `assets/motion/_utils/harmony-check.js` line 59-63
```js
return {
  accentArea: accentArea,
  totalArea: viewportArea,        // ❌ 返回 viewportArea
  ratio: totalArea > 0 ? accentArea / totalArea : 0  // ❌ ratio 用 totalArea
};
```
**问题**: `accent.totalArea` 字段返回的是 `viewportArea` (window.innerWidth × innerHeight), 但 `ratio` 实际计算用的是循环累计的真实 `totalArea` (所有可视元素面积之和). **外部使用 `report.accent.totalArea` 的人会被误导**.

**修复**: 统一 — `ratio = accentArea / viewportArea` (符合 60-30-10 法则语义), `totalArea` 也改为返回真实 totalArea, 或删除 `totalArea` 字段.

### BUG-02 · attention.js:155 · rootMargin 方向反
**位置**: `assets/motion/_utils/attention.js` line 154-156
```js
observer = new IntersectionObserver(function (entries) {
  // ...
}, {
  threshold: 0.1,
  rootMargin: '0px 0px -10% 0px'    // ❌ 方向反了
});
```
**问题**: `rootMargin` 是 **shrinks** root rectangle (减小观察区域). `'0px 0px -10% 0px'` 表示底部边界向**内**收缩 10%, 即元素**要进入视口顶部 90% 范围**才算 intersecting. 这导致元素到达底部 10% 视口时**不触发**, 反而延后触发, 视觉效果是"卡顿".

**修复**: 改为 `'0px 0px 10% 0px'` 或 `'50px 0px'` (提前触发), 让元素在进入视口前 10% 就开始动画.

### BUG-03 · components.css · .progress-bar 缺 width: 0
**位置**: `assets/design/components.css` `.progress-bar` 块
```css
.progress-bar {
  position: absolute;
  inset: 0 auto 0 0;      /* ❌ 缺 width 默认值 */
  background: var(--accent);
  border-radius: 999px;
  transition: width 0.4s cubic-bezier(0.2, 0, 0, 1);
}
```
**问题**: `inset: 0 auto 0 0` 展开为 `top:0; right:auto; bottom:0; left:0`. 配合 `position: absolute`, 元素的 width 默认是 `auto`, 而 absolute 元素 auto width = **包含块的宽度** (整个 progress 容器). 所以 progress-bar **默认占满**, 而 JavaScript 设置 `width: 50%` 时是从 100% → 50% 动画, 而非 0% → 50%.

**修复**: 加 `width: 0;` 默认值.

### BUG-04 · components.css · skeleton 闪烁
**位置**: `assets/design/components.css` `.skeleton` keyframes
```css
.skeleton {
  background: linear-gradient(90deg, var(--bg-secondary) 0%, oklch(95% 0.008 250) 50%, var(--bg-secondary) 100%);
  background-size: 200% 100%;
  animation: skeleton-shimmer 1.5s ease-in-out infinite;
}
@keyframes skeleton-shimmer {
  0% { background-position: 200% 0; }
  100% { background-position: -200% 0; }   /* ❌ 循环时跳变 400% */
}
```
**问题**: 关键帧从 `200%` 到 `-200%`, 移动距离 400%. 动画 `infinite` 循环时, 从 `-200%` 瞬间跳回 `200%`, **用户能看到一次闪烁** (从右到左结束后, 突然跳到右, 然后再向左).

**修复**: 改为 `100% { background-position: -100% 0; }` (移动 300%, 跳变 100%, 但因为背景色是对称渐变, 跳变不明显). 或改 `0% { background-position: 100% 0; }` 让首尾状态一致.

## 🟡 P1 应该修 bug (3 个)

### BUG-05 · attention.js:96 · delay 解析静默失败
**位置**: `assets/motion/_utils/attention.js` line 89-103
```js
var match = delayStr.match(/calc\(([\d.]+)(?:ms|s)?\s*\*\s*(\d+)\)/);
if (match) {
  // ... calc 解析
} else {
  var simple = parseFloat(delayStr);   // ❌ 对 "calc(80ms * 1)" 返回 NaN
  if (!isNaN(simple)) { /* ... */ }
  // else 静默 delay = 0
}
```
**问题**: 当 `getComputedStyle` 返回 `calc(80ms * 1)` 且正则匹配失败 (edge case: 浏览器输出格式略有不同, 例如 `calc(80ms * 1) ` 带尾空格, 或 `calc( 80ms * 1 )` 带空格), `parseFloat` 返回 NaN, **静默 delay = 0, stagger 完全失效但不报错**.

**修复**: 加 console.warn 提示 + 改进正则容忍空格.

### BUG-06 · harmony-check.js:38 · critical 误计 accent 面积
**位置**: `assets/motion/_utils/harmony-check.js` line 38
```js
var accentEls = root.querySelectorAll('[class*="accent"], [class*="bg-accent"], .btn-primary, [data-attn="critical"]');
```
**问题**: `[data-attn="critical"]` 在 `attention-hierarchy.css` 中定义为 `color: var(--text-primary)` (黑色), **不是 accent 色**. 但 selector 把所有 critical 都当作 accent 色计算面积, 导致 accent.ratio 虚高.

**修复**: 移除 `[data-attn="critical"]`, 改为只计 `.bg-accent`, `.btn-primary`, `[class*="accent-bg"]`, `[class*="accent-text"]`.

### BUG-07 · harmony-check.js:88 · reveal.js 隐藏 slide 被误计
**位置**: `assets/motion/_utils/harmony-check.js` line 85-89
```js
function audit(root) {
  root = root || document.body;     // ❌ 整个 document.body
  var accent = countAccentArea(root);  // ❌ 包括所有 reveal slides (隐藏的)
  // ...
}
```
**问题**: reveal.js 的所有 `<section>` 都在 DOM 中, 但同一时刻只有一个 slide 可见. audit 默认从 `document.body` 统计, **把所有 13 个 slide 的 accent 元素都计入**, 误报.

**修复**: 默认 root 改为当前可见 slide, 例如 `document.querySelector('.reveal .slides > section.present') || document.body`.

## 🟢 P2 可改进 (1 个)

### BUG-08 · attention-hierarchy.css:117 · CTA 唯一性无 opt-out
**位置**: `assets/design/attention-hierarchy.css` `.btn-primary ~ .btn-primary` 块
```css
.btn-primary ~ .btn-primary {
  background: var(--bg-secondary);
  color: var(--text-primary);
  /* ...降级为 secondary */
}
```
**问题**: 用户**有意**写两个 primary CTA (例如双 CTA 行 "立即试用 + 查看演示"), CSS 强制降级违反用户意图. 没有 opt-out 机制.

**修复**: 添加 `[data-cta-multi="true"] .btn-primary` 例外, 或 `[data-cta-unique="false"]` 关掉守则.
