## 步骤 2：拆解到基本事实（公理化）

### 4 条不可违反的公理

**公理 1：用户真正想要的是"一个能直接发布的短视频"**
- 不是 txt 提示词
- 不是校验报告
- 是 .mp4 文件，能上抖音/视频号

**公理 2：用户的时间/精力是稀缺资源**
- 短剧编剧 1 周产 1 集
- 不可能手动维护每个镜的资产
- 不可能跑 4 个工具拼起来

**公理 3：AI 视频生成成功率 < 100%**
- 即梦单镜成功率 60-80%
- 必须有"失败重做"机制
- 用户的耐心 < 3 次重做

**公理 4：跨集一致性是商业项目的硬要求**
- 霸总剧要拍 30 集
- 刘一鸣的脸不能每集变
- 服装/场景/道具也要一致

## 步骤 5：5 大偏差清单

### 偏差 1：错把"提示词"当"产品" 🔴 P0

**症状**：
- skill 自我定位："通用分镜头提示词生成器"
- 输出 12 个 txt + 校验报告
- 视频生成、拼接、字幕完全没做

**真正的产品**：
- 用户要的是 .mp4
- 提示词是中间产物（占 50% 工作量）
- 后续 50% 还是手工

**怎么修**：
- 增加 M5（视频生成）+ M6（拼接）模块
- 接入即梦 API / 可灵 API
- 端到端一条命令

---

### 偏差 2：错把"文档化"当"工程化" 🔴 P0

**症状**：
- SKILL.md 35K / 4 个 modules / 10 个 KB / 8 个 STYLE / 2 个 T
- 真实可执行工具：4 个 Python + 2 个 Bash
- 文档:工具 = 90:10

**问题**：
- LLM 读 35K 文档 → 信息过载
- 真正做事时找不到关键约束
- 文档沉淀 ≠ 工程沉淀

**怎么修**：
- 文档:工具 = 50:50
- 关键约束写在工具里（hardcode）
- KB 文档是参考，不是必读

---

### 偏差 3：错把"防御"当"主线" 🟡 P1

**症状**：
- 23 项硬约束
- 6 角色投票机制
- M15 元注释检测
- 失败=0 原则

**问题**：
- 防御做得好 = 问题少
- 但内容创作（画面描述）质量才是 LLM 工作
- 用户真正关心的是"生成的视频好不好看"，不是"约束有几条没违反"

**怎么修**：
- 约束从 23 项收敛到 5-8 项（核心项）
- 增加"内容质量评估"维度（CLIP 相似度 / 物理一致性）
- 减少"合规自检" 占比

---

### 偏差 4：错把"单项目"当"长期" 🟡 P2

**症状**：
- 完美支持 EP01 单集
- EP02/EP03 怎么做？人物档案怎么复用？
- 知识沉淀了（KB01-10），但资产没沉淀

**问题**：
- 商业项目要拍 30 集
- 第 2 集刘一鸣的脸不能变
- 手动维护 30 集 × 11 镜 = 330 个资产 → 不可持续

**怎么修**：
- 引入 IP-Adapter / LoRA 持久化角色
- 角色库 + 场景库 + 道具库 跨项目复用
- 等 IP-Adapter 成熟再做

---

### 偏差 5：错把"启发式"当"LLM 增强" 🟡 P1

**症状**：
- m1_parse.py 用关键词检测风格（霸总/总裁/霸道 → 古早霸总）
- m2_design.py 用规则分配镜数（每镜 8 秒 × 80 秒 = 10 镜）
- m3_generate.py 用模板生成 6 段

**问题**：
- 启发式 vs LLM 增强：LLM 更准
- 启发式 1 镜 0.001s，LLM 1 镜 3s
- 但 1 集 11 镜 × 0.5 元/次 = 5.5 元
- 启发式省钱但不准

**怎么修**：
- M1/M2 用 LLM 增强（准确度高，~5 元/集）
- M3 启发式 + 模板（成本敏感）
- M4 启发式（用 validate_prompt.py 即可）

## 核心判断

**skill 的"提示词工程"部分做得很扎实**：
- AI 一次成功率 5-15% → 80-95%（提升 6-8 倍）✅
- 物理细节 356→≤96（-73%）✅
- 元注释零残留 ✅

**但 skill 的"产品定义"偏离了 60%**：
- 应该是"端到端 AI 视频生成器"
- 实际是"提示词生成器 + 23 项约束 + 校验报告"
- 中间产物优化到极致，但端到端缺一半

**下一步建议**：
1. **不要继续优化提示词工程**（已经够好）
2. **开始做 P0：端到端 pipeline**（m5 + m6 + run_skill 升级）
3. **降低文档比例**（把 KB 关键约束搬到工具）
4. **减少约束数量**（23 → 5-8 项核心约束）
