## 【WHY 引擎】苏格拉底 6 步追问"真假实现"

### 步骤 1：理解问题
"真假实现"是什么意思？
- 假实现 A：代码存在但跑不通
- 假实现 B：代码跑通但产生的结果**不是用户想要的**
- 假实现 C：演示好看但**没有真实环境验证**
- 假实现 D：单元测试通过但**端到端失败**

### 步骤 2：揭示假设
- 假设 1：跑过测试 = 实现 ✅ → **假**（M5/M6 跑过但只产生 mock 文件）
- 假设 2：能跑通 = 能交付 → **假**（mock 视频不算交付物）
- 假设 3：功能多 = 价值大 → **假**（3 个假实现 = 0 价值）

### 步骤 3：检验证据 — 三大铁证

| 证据 | 真相 |
|------|------|
| **M5 生成的"视频"** | 实际是 `*.mp4.mock` 文件，不是真视频 |
| **M6 拼接的"EP01_final.mp4"** | 在无 ffmpeg 时**根本不存在**，只有 ffmpeg.sh 脚本 |
| **m10_inherit 跨集输出** | 注入"延续资产"块，但**没 EP02 验证资产真的一致** |

### 步骤 4：换位思考
**用户视角问 3 个问题**：
1. 我能下载你的 skill 跑出真视频吗？→ ❌ 不能（除非我有即梦/可灵 API key + ffmpeg）
2. 我能跨集做 EP02 保持角色一致吗？→ ❌ 不知道（没真跑过）
3. 我能用 LLM 提升准确度吗？→ ❌ 不知道（没真用 LLM 跑过）

### 步骤 5：追踪推论
- 假实现 → 用户拿到 skill 后**第一个跑通就放弃**
- 推论 5 步：用户用一次不用 = 60% → 假实现是这 60% 的主因

### 步骤 6：反思问题
"为什么我们做的是假实现？"
- 没真用户逼着"必须能跑出视频"
- mock 模式是"工程完整性"考虑，但被误用为"假完成"
- **没有交付物思维** — SpaceX 火箭要么飞起来，要么没飞

## 【HOW 引擎】第一性原理 5 步 — 拆解"实现"的本质

### 步骤 1：识别假设
"skill 是代码，代码 = 实现" → 拆解

### 步骤 2：拆解到基本事实
**skill 的本质 = 流水线**：
```
剧本 → M1 → M2 → M3 → M4 → M5 → M6 → 视频
                                        ↑ 真假分水岭
```

**基本事实**：
- M1-M4：纯文本/JSON 处理 ✅（真实现）
- M5：调用第三方 API → 真视频文件（**未真实调用过**）
- M6：ffmpeg 拼接 → 完整 mp4（**未真实拼接过**）

### 步骤 3：5 Whys
```
Q1: 什么叫"实现"？
A1: 用户输入剧本，30 分钟拿到 1 集完整视频 mp4

Q2: 现在能吗？
A2: ❌ 不能，因为 M5 是 mock、M6 是脚本

Q3: 为什么是 mock？
A3: 1) 没 API key  2) 没 ffmpeg  3) "先 mock 后真跑"是工程惯例

Q4: "先 mock 后真跑"是借口吗？
A4: 部分是借口。SpaceX 第 1 次发射是真火箭，不是 mock

Q5: 真问题是什么？
A5: skill 的"工程完整性"和"用户可用性"是两件事
    工程完整 ≠ 用户可用
    mock 保证工程完整，但用户拿到的是**假的视频文件**
```

### 步骤 4：从零重建
**如果从零开始，skill 的最小可用定义（MUD）**：
- 用户输入：1 集剧本
- 用户输出：**1 个能播放的 mp4 文件**
- 时间：30 分钟内
- 质量：能看出是什么故事

**当前距离 MUD 的差距**：
- M5 mock → 0%（不产生视频）
- M6 ffmpeg 脚本 → 50%（脚本可能对，但没验证）
- M3 prompt → 70%（可能不准但能跑）

### 步骤 5：验证可行性
**真实现 M5/M6 的 3 个必要条件**：
1. 真实 API 凭证（即梦/可灵）→ **成本 ¥30-100/集**
2. ffmpeg 安装 → **用户自己装**
3. 端到端跑通 1 集 → **1-2 小时**

**结论**：可行。**没做 = 优先级问题，不是技术问题**。

## 关键洞见

### 洞见 1："工程完整性" ≠ "用户可用性"
- mock 保证流水线不中断
- 但用户**不知道**也不在乎 mock
- 用户要的是"30 分钟真视频"

### 洞见 2：假实现 = 信任破产
- 用户试一次发现假 → 永久流失
- 比"没功能"更糟（"有功能但假" = 0 - 100 = -100）

### 洞见 3：真实度 = 67%
- 3 大假实现修完 → 真实度 67% → **92%**
- 1.5 天补完半实现 → 真实度 **97%**
- 之后才轮到"新功能"（E2 等）

### 洞见 4：成本/价值 ROI
- P0-1（M5 真跑）：1-2 天 → +500 价值 → **最高 ROI**
- P0-3（EP02 跨集）：2 天 → +300 → 高 ROI
- E2（m2 LLM）：1 天 → +40 → 低 ROI
- 写文档：1 天 → +35 → 低 ROI
