## 步骤 3：1800-1900 目标合理性分析

### 3.1 用户理由（推测）

1. **AI 视频 API 注意力分散**：即梦/可灵 > 2000 字后关键指令被淹没
2. **v3.4 + v3.5 治本后信息密度已经很高**：再保留 2000+ 字 = 冗余
3. **短视频单镜匹配**：5-8 秒单镜不需要 2400 字那么长
4. **API token 成本**：单镜省 350 字 × N 镜 = N×350 token 节省

### 3.2 1800-1900 的优势

| 维度 | 2000-2400 | 1800-1900 |
|------|-----------|-----------|
| AI 关键指令突出度 | ⚠️ 中 | ✅ 高 |
| Token 成本 | 基线 | -16% |
| 单镜时长匹配度 | 6-12s | 5-8s |
| 物理细节完整度 | ✅ 高 | ⚠️ 中（需重测） |
| 风格参考完整度 | ✅ 3-5 部 | ⚠️ 2-3 部 |

### 3.3 风险评估

| 风险 | 严重度 | 缓解 |
|------|--------|------|
| KB07 标注 1500-2000 为"信息密度低" | 🟡 中 | 同步修改 KB07 + 实测验证 |
| 物理细节被压缩（动作链/肌肉物理） | 🟡 中 | 重点保证画面描述段 700+ 字 |
| AI 一次成功率可能下降 | 🟡 中 | 用 EP01 26 镜实测对比 |
| 风格参考只能保留 2-3 部 | 🟢 低 | 当前 KB07 已建议 3 部以内 |
| 跨镜衔接可能丢失细节 | 🟢 低 | 衔接由 M4 校验，不在字数 |

## 步骤 4：从零重建 — 优化方案

### 4.1 字数阈值重设（3 处代码）

```python
# validate_prompt.py · continuity_check.py · check_word_count

elif count < 2000:  # 偏少
```

## 真实价值 vs 真实风险

### 价值（量化）

| 维度 | 量化收益 |
|------|----------|
| 单镜字数 | -350 字（-16%）|
| 26 镜 EP01 总字数 | -9100 字 |

| 跨镜衔接丢失 | 道具状态描述变薄 | 道具演化由 M4 + 资产库双校验 |

## 决策建议

### 推荐：分阶段实施

1. **先小范围试点**：用 EP01 26 镜中 3 镜（开场/中段/收束）跑 v3.6 阈值，看 AI 一次成功率
2. **若试点通过**：扩展到全 26 镜，对比 v3.5 vs v3.6
3. **若对比通过**：写入 v3.6 治本 + KB12
4. **若任一步失败**：回滚到 v3.5 阈值，分析失败原因再优化

### 不推荐：直接全量切换

- v3.4 + v3.5 治本经验已沉淀，全量切换风险高
- 26 镜实测数据是 skill 的护城河，不能轻易破坏
