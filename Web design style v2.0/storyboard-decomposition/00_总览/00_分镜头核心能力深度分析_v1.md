# skill 分镜头核心能力深度分析 v1 — 第一性原理

> **核心问题**：**剧本 → 分镜头** 这个根本任务，skill 现在真正能做什么？做到什么程度？
> **方法**：第一性原理 5 步 + 拆解到不可再分的基本事实
> **结论**：skill 在分镜头领域**已建立护城河**（v3.4/v3.5 治本 80-95% AI 一次成功），但有 4 个真实边界

---

## 步骤 0：纠正跑偏

**之前我犯了什么错**：
- 把"端到端视频生成"当作 skill 核心
- 反复讨论 M5/M6 假实现（视频 API + ffmpeg）
- **完全忽略**：用户问的是"剧本拆分分镜头"，不是"生成视频"

**真实任务边界**：
- 输入 = 剧本（小说/大纲/分场/完整剧本）
- 输出 = **分镜头表**（每镜的：景别/运镜/动作/口播/光影/技术参数）
- **不包含** = 真视频生成、拼接、字幕压制

**正确的"分镜头完成"定义**：
> 一份能直接被 AI 视频 API（即梦/可灵）接受的提示词 txt，包含 v3.4 治本 + v3.5 元注释剥离 + 5 段式结构 + 资产引用。

## 步骤 1：识别"分镜头"假设

| 隐含假设 | 真实性 |
|----------|--------|
| 分镜头 = 镜数列表 | ⚠️ 真，但不够（缺景别/运镜/物理细节） |
| 分镜头 = 提示词 | ⚠️ 提示词是分镜头的实现形式（v3.5） |

| 分镜头 = 包含衔接 | ✅ 跨镜一致性（M4 校验） |
| 分镜头 = 跨集一致 | ✅ 资产库 + m10 跨集继承 |

## 步骤 2：拆解到基本事实 — 分镜头的不可再分元素

**一个完整分镜头 = N 个镜，每个镜含 6 段**（v3.4 治本）：

```
1. 场景空间坐标（机位/角色坐标/位移/道具状态）
2. 资产引用（人物/场景/道具·库引用）
3. 人物资产档案（v3.4 治本·参考图已承载则不重复）
4. 镜头参数（时长/画幅/景别变化/焦距/运镜）
5. 画面描述（时间分段+动作链+口播+前景背景关系）
6. 光影氛围 + 技术参数（色温/光比/T-stop/景深）
```

**质量标准**（v3.5 + M15）：
- 字符数：2200±200 字
- 无元注释（无"决策："/"修复："/"规则："段首）
- 物理细节精确到 0.5 秒/动作链/肌肉真实物理
- 资产引用齐全（人物/场景/道具）
- 工程关键词齐全（T-stop/K 色温/景深/焦距）
