## 4 引擎分析

### 【WHY 引擎】苏格拉底 6 步追问

#### 步骤 1：理解问题
"用户说'下一步'是什么意思？"
- A 解释：继续做更多功能（工程视角）
- B 解释：让 skill 真正被用起来（产品视角）
- C 解释：发版本/写文档/做教程（传播视角）

**歧义点**：用户没说"下一步做什么"，只说"下一步"。

#### 步骤 2：揭示假设
- 假设 1：现在还差功能 → 真的吗？E1 后 11 个脚本 + 3 个库 + 1 验证器，**已超最小可用产品（MVP）**
- 假设 2：做更多功能 = 价值 → 第一性原理证伪：SpaceX 第 4 次火箭发射才成功，**第 1-3 次是失败**
- 假设 3：技术问题 = 最重要问题 → 苏格拉底：是不是"没人用"才是？

#### 步骤 3：检验证据
- 证据 A：P0/P1/P3/E1 全部跑通测试 ✅
- 证据 B：但**没有任何真实用户用过 30 天** ❌
- 证据 C：v1 风险分析已识别 60% 概率"用一次不用"——但未验证 ❌
- 证据 D：Mr.Lin 单人开发，无 PMF 验证能力 ⚠️

**结论**：技术证据已足够，**用户行为证据 = 0**。

#### 步骤 4：换位思考
- 霸道总裁剧本创作者视角："这个 skill 跑完 1 集 80s 视频要多久？值得吗？"
- 短剧 UP 主视角："30 分钟真能跑出 1 集吗？质量呢？"
- 答案：**没人回答过这些问题**，因为**没真用户试过**。

#### 步骤 5：追踪推论
- 如果继续做 E2 m2 LLM → skill 解析更准 5% → **谁在乎？没有用户用**
- 如果做 P0.5 真实跑通 → 暴露 3 个真实痛点 → 指导**所有**未来工作
- 推论 5 步：技术完美 + 没人用 = 价值 0

#### 步骤 6：反思问题
- "用户为什么问下一步？"
  - 可能 A：想"做更多"（创造满足感）
  - 可能 B：想"做完收工"（任务完成感）
  - 可能 C：想"做对的事"（避免浪费）

**反问**：E1 刚做完，"做对的事" vs "做更多的事"哪个优先？

### 【HOW 引擎】第一性原理 5 步

#### 步骤 1：识别假设
"skill 是工具集" → 拆解

#### 步骤 2：拆解到基本事实
- skill 输出 = 26 镜 × 5 秒 视频 = 130s 短视频
- 视频生成 = M3 prompt → 视频 API → mp4
- **基本事实**：1 集视频制作 = 真人 3-7 天；skill 承诺 = 30 分钟
- **真问题**：用户愿不愿意用 30 分钟换 3 天？

#### 步骤 3：5 Whys
```
Q1: 用户会不会用 30 分钟换 3 天？
A1: 不知道，从未验证

Q2: 为什么没验证？
A2: 1) 没找用户  2) 怕验证失败  3) 觉得技术再完善点就能验证

Q3: 怕验证失败说明什么？
A3: 缺乏"如果失败就调整"的韧性 — Musk 反例：3 次失败才第 4 次成功

Q4: "技术再完善点"是借口吗？
A4: 60% 是借口 — 永远可以"再完善一点"

Q5: 真问题是什么？
A5: 不是技术是否够好，而是"用户视角的 30 分钟体验"长什么样
```

#### 步骤 4：从零重建
- 假设今天 skill 是空白的，只有 1 个能力可用
- 选什么？→ **"30 分钟真能跑出 1 集视频"**
- 为什么？→ 这是用户唯一关心的事

#### 步骤 5：验证可行性
- 物理可行：已有 run_skill.sh + EP01 测试剧本
- 成本可行：API 费用 ¥10-30 跑 1 集
- 时间可行：1-2 小时
- 必要条件：**Mr.Lin 亲自当用户，跑 1 遍**

## F0 真跑执行计划

### Phase 1: 准备（30 分钟）
- 选 1 个真实剧本（已存在：EP01-完整分镜头脚本.md）
- 检查环境：API key / ffmpeg / 视频 API 凭证
- 决定 mock 模式 vs 真实模式

### Phase 2: 跑（30 分钟）
- 一键执行 `run_skill.sh --to-video --mock-llm`
- 真实记录每一步的：
  - 时间
  - 困惑点
  - 错误
  - 期望 vs 实际

### Phase 3: 报告（30 分钟）
输出"F0 真跑报告 v1"含：
- 端到端耗时表
- 用户视角 Top 3 痛点
- Top 3 "我希望有的功能"
- Top 1 立即可做的小改进

### 决策门
- 痛点数 ≥ 3 → 启动 P2 痛点修复（不是 E2）
- 痛点数 < 3 → 启动 E2 m2 LLM 增强
- 跑不通 → 立即修
