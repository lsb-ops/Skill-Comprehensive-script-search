# SKILL: 通用分镜头提示词生成器（v3.7 治本）

> 从任意剧本生成 **≤1800 字** 的 AI 视频分镜头提示词（支持即梦/可灵/Sora/Runway）
> **v3.7 新增**：字数约束 2000-2400 → **≤1800**；加入【声音_角色：】模块（4 维度音色/语速/共鸣/风格）

---

## 一、用途

将任意格式的剧本（小说/故事大纲/分场剧本/完整剧本）转化为可直接喂给 AI 视频生成工具的分镜头提示词，每个提示词 **≤1800 字**（v3.7 治本），结构化、可执行、含**声音参考**模块。

**适用人群**：
- 短剧/短视频编剧
- AI 视频创作者
- 影视分镜师
- 内容创作团队

**输入**：任意剧本
**输出**：N 个分镜 txt 文件 + 场景主控档案 + 校验报告

---

## 二、输入规范

| 参数 | 必填 | 默认值 | 说明 |
|---|---|---|---|
| 剧本原文 | ✅ | - | 任意格式（小说/大纲/分场/完整剧本） |
| 风格 | ❌ | 自动检测 | 古早霸总/都市情感/古装江湖/科幻未来/校园青春/悬疑推理/战争军事/家庭伦理 |
| 总时长 | ❌ | 80 秒 | 短视频默认 60-90 秒 |
| 画幅 | ❌ | 9:16 竖屏 | 9:16/16:9/1:1 |
| AI 平台 | ❌ | 即梦 | 即梦/可灵/Sora/Runway |
| 集数/EP | ❌ | EP01 | 用于输出文件命名 |

| `01_分镜头/` | N 个分镜 .txt | ✅ skill 自动 | ⭐⭐⭐⭐⭐ AI 唯一输入 |

| 动作 | skill 是否执行 |
|---|---|
| 创建 `{剧本名}/` 文件夹 | ✅ 是 |
| 创建 `00_总览/` 并写入 M1/M2/M4 报告 | ✅ 是 |

### 3.5 剧本名提取规则

从输入文件名自动清理：
- ✅ 去除：`副本`、`(数字)`、`.docx`、`.txt`、`.md` 等后缀
- ✅ 替换：空格 → `_`
- ✅ 保留：剧本核心名 + EP 编号

**示例**：
| 输入文件名 | 提取后剧本名 |
|---|---|
| `副本当霸道总裁拥有媒体素养 EP01(3).docx` | `副本当霸道总裁拥有媒体素养_EP01` |
| `灰姑娘嫁入豪门 EP05.txt` | `灰姑娘嫁入豪门_EP05` |
| `final_script_v3.docx` | `final_script_v3` |

## 四、4阶段工作流

### 阶段0：初始化输出目录（强制前置步骤）
```
输入：剧本文件名 + output_root
任务：
  0.1 从文件名提取剧本名（清理"副本"、"(N)"等）
  0.2 创建剧本名根目录：{output_root}/{剧本名}/
  0.3 创建 5 个子文件夹：
       - 00_总览/         （后续 M1/M2/M4 写入）
       - 01_分镜头/       （后续 M3 写入）
       - 02_场景主控/     （空，用户手动放图）
       - 03_人物档案/     （空，用户手动放图）
       - 04_道具档案/     （空，用户手动放图）
输出：5 个文件夹的骨架（02/03/04 为空）
```

**实现示例（任何语言）：**
```python
import os, re
script_name = re.sub(r'\(\d+\)|\.docx$|\.txt$', '', input_filename)
script_name = re.sub(r'\s+', '_', script_name.strip())
base = f"{output_root}/{script_name}"
for sub in ['00_总览', '01_分镜头', '02_场景主控', '03_人物档案', '04_道具档案']:
    os.makedirs(f"{base}/{sub}", exist_ok=True)
```

### 阶段1：剧本解析（调 M1）
```
输入：剧本原文
输出位置：{output_root}/{剧本名}/00_总览/00_M1_剧本解析.md
任务：
  1.1 场景切分（时间+地点）
  1.2 人物识别（数量+名字+角色定位）
  1.3 5幕结构识别（建制/触发/反转一/反转二/收束）
  1.4 剧本原词 100% 提取（对白+舞台指示）
  1.5 戏剧动作清单
输出：场景列表+人物清单+5幕结构+剧本原词+动作清单
```

### 阶段2：分镜设计（调 M2 + 风格预设）
```
输入：阶段1输出 + 风格预设（styles/STYLE_xxx.md）
输出位置：{output_root}/{剧本名}/00_总览/00_M2_分镜设计.md
任务：
  2.1 加载风格预设（视觉基线+CUE模式+典型结构）
  2.2 镜数规划（按风格典型结构）
  2.3 时长分配（每镜 5-12 秒）
  2.4 CUE 设计（按风格 CUE 模式）
  2.5 色温曲线设计
  2.6 场景主控档案生成（每个场景 1 个，作为 M2 报告的子节）
  2.7 衔接表生成（上镜结束位置+本镜开始位置）
输出：分镜大纲+场景主控档案+衔接表+CUE清单+色温曲线
```

### 阶段3：提示词生成（调 M3 + T1 模板 + 反复比对）
```
输入：阶段2输出 + 剧本原文 + 上镜 TXT + 场景拓扑图
输出位置：{output_root}/{剧本名}/01_分镜头/镜{NN}_{主题}.txt × N
任务：
  3.0 读取 4 源（剧本 / M2 / 上镜 / 拓扑）——强制
  3.1 按 T1 模板逐镜生成
  3.2 套用 6 段式结构（资产引用 / 参考图引用 / 人物资产档案 / 镜头参数 / **场景空间坐标** / 画面描述 / 光影氛围 / 技术参数 / 音效 / 台词）
  3.3 不变量检查（8 项）——强制
  3.4 严格 ≤1800 字控制（合并版 ≤1700 · v3.7 治本）
  3.5 保留所有剧本原词
  3.6 资产标记与图片同名（人物_XXX / 道具_XXX / 场景_XXX）
  3.7 反复比对 3 轮（剧本 vs TXT / 跨镜一致性 / AI 可执行性）——强制
  3.8 凭空补完清单 + 用户确认——强制
输出：镜XXX_xxx.txt × N（含自检报告）
```

### 阶段4：衔接校验（调 M4）
```
输入：{output_root}/{剧本名}/01_分镜头/*.txt（所有镜）
输出位置：{output_root}/{剧本名}/00_总览/00_M4_衔接校验报告.md
任务：
  4.1 站位一致性（180 度轴线）
  4.2 道具轨迹（跨镜位置）
  4.3 CUE 节奏（时间点+效果）
  4.4 字数校验（≤1800 字 · v3.7 治本）
  4.5 风格一致性（色温/光比/视觉参考）
输出：校验报告（issue 清单+修复建议）
```

### 阶段5：用户图片填充（手动，非强制）
```
位置：{output_root}/{剧本名}/02_场景主控/, /03_人物档案/, /04_道具档案/
任务：用户根据 01_分镜头/ 中的资产标记，在对应文件夹放入同名图片
  - 【人物_刘一鸣：】 → 03_人物档案/人物_刘一鸣.png
  - 【道具_墨镜：】   → 04_道具档案/道具_墨镜.png
  - 【场景_弄堂内：】 → 02_场景主控/场景_弄堂内.png
用途：在即梦/可灵中 @ 引用，保证人物/道具/场景一致性
```

## 六、核心约束（不可违反）

1. **字数**：每镜 **≤1800 字**（v3.7 治本）
2. **结构**：5 段式（资产引用/镜头参数/画面描述/光影氛围/技术参数/音效/台词/合规）
3. **原词**：剧本原词 100% 保留，不改一字
4. **站位**：180 度轴线规则
5. **CUE**：3 次戏剧断点（按风格）
6. **字幕**：画面中不出现任何文字，全部后期 PR/AE 叠加
7. **图片资产**：通过【】资产引用锚定，不在提示词中描述人物外貌
8. **v3 提示词工程规范**（KB08 强制约束）：
   - `v3.0强化` 单镜 ≤ 2 次
   - `真实物理`（抽象词）≤ 5 次，必须替换为具体物理数字
   - `真实物理引擎渲染`（即梦 hot-token）每镜 1-2 次保留
   - 每镜必加 T-stop 值（T2.8 / T1.4 等）
   - 抽象词黑名单（蒙太奇美学/教化者美学/冷静科普美学/极具压迫感/戏剧光晕/AI 后期逆光晕/AI 渲染/电影级/8K/HDR/杜比视界）
   - 风格参考 ≤ 3 部，格式 `风格参考：X；副参考：Y`
   - 色温与时间/天气匹配（日景 5000K+ / 黄昏 3000-3500K / 冷科普 4000K / 升华 5500K）
   - Halation 日常对话戏关闭，CUE③ 升华段开启
   - 物理数字 ≥ 5 个（G=-9.8 m/s² / 0.3s 滞后 / 1.5米每秒 / 45° / Hertz 0.02s）
   - 真人感锚点词 ≥ 3 个（皮肤微血管/发根阴影/衣料重力褶皱/Hertz 接触）
9. **反复比对循环**（M10 强制）：M3 必须执行 3 轮自检（剧本 vs TXT / 跨镜一致性 / AI 可执行性），任一警告 → 回到步骤 1（最多 3 次循环）
10. **凭空补完确认**（M10 强制）：M3 输出前必须列出"剧本未提及"清单，**用户必须显式确认**才输出最终 txt
11. **场景拓扑引用**（M10 强制）：每镜 prompt 顶部必须含"场景空间坐标"段（含 3 人物坐标 + 机位方向 + 上镜→本镜位移 + 道具状态）
12. **不变量检查**（M10 强制）：M3 步骤 9.5 强制 8 项（物理细节 ≤ 8 / 手部微动作 ≤ 3 / 文字锚点 = 0 / 场景空间坐标存在 / 机位方向标注 / 物理量级全 / 镜头 12 参数全 / 凭空补完警告）
13. **6 角色调用机制**（M11 强制）：关键决策点（合并/拆分/超字数/失败复盘）必须调用 6 角色（导演/摄影/剪辑/AI专家/编剧/制片）+ 1 主持人讨论，禁止单视角决策
14. **决策日志记录**（M11 强制）：每次合并/拆分决策必须记录到 `00_总览/00_决策日志.md`（含 6 角色立场 + 加权投票 + 综合决策）
15. **角色权重固定**（M11 强制）：AI 专家 0.30（最高）/ 导演 0.20 / 摄影 0.15 / 剪辑 0.15 / 编剧 0.15 / 制片 0.05 — 反映"AI 生成 + 艺术性 + 经济性"三角平衡
16. **时长分级决策**（M12 强制）：按幻觉率曲线分级——≤5s（高成功）/ 6s（标准）/ 7-8s（需强约束）/ 9-10s（强制拆分）/ 11-15s（仅允许叠化合并）/ >15s（禁止）。M3 步骤 9.7 必须按此分级决策
17. **反幻觉 9 大规则**（M12 强制）：M3 步骤 9.6 必须自检——①单一动作原则 ②物理量级强制 ③资产标记替代外貌 ④场景空间坐标前置 ⑤风格参考≤3部 ⑥抽象词黑名单 ⑦CUE时刻物理跳变（不渐变）⑧禁止反作用力/抛物线/帧率切换 ⑨同镜同景别
18. **失败=0 原则**（M12 强制）：成功率< 70% 的方案禁止输出，宁可拆分/叠化也不冒险。M4 失败复盘必须触发 6 角色机制（M11 时机 5）。**绝不追求快，只追求对**
19. **物理细节按时长分级**（M13 强制）：≤5s ≤4 / 6s ≤6 / 7-8s ≤8 / 9-10s ≤8（强约束）/ 11-15s ≤8（叠化合并，6角色一致同意）。M3 步骤 9.8 必过
20. **删除"补充物理细节"段**（M13 强制）：关键物理融入对应段落（画面描述/音效/技术参数）。禁止独立段（信号稀释、信噪比下降）
21. **反作用力/抛物线/帧率切换/微距离黑名单**（M13 强制）：
    - ❌ "反作用力 0.2 米" → 改写"肩膀相撞肩部接触面 5cm²"
    - ❌ "45° 抛物线 v₀=2 米/秒" → 改写"喷出 0.5 米"
    - ❌ "24→12fps" → 改写"相撞瞬间周围人物动作变慢"
    - ❌ "镜腿分离 0.05/0.1 秒" → 改写"镜腿分离"
    - ❌ "眉头松+下颌垂+瞳孔缩" → 合并"面部肌肉松弛"
    - ❌ "领带钟摆 ±3°" → 改写"领带随呼吸轻微钟摆"

22. **M11 决策标注**（M13 强制）：每镜标题必含 `M11决策#XXX` 决策号标注，反向追溯决策原因
23. **资产档案去重·v3.4 治本**（M14 强制·NEW）：人物资产档案段不得重复描述参考图已承载信息（年龄/身高/服装/发型/表情/道具外观/场景细节），全部由参考图承载；仅可保留合规/禁忌等图片无法承载的信息。M3 步骤 9.9 必过 + validate_prompt.py v2.1 约束 23 自检

## 七·v3.4 治本（M14·资产档案去重）

### 7.1 问题本质

**用户洞察**："由于我已经给了参考图片，关于人物的描述已经不重要了"

**核心矛盾**：提示词文字描述 vs 参考图视觉信息 = 两个独立信号源承载同一信息
- AI 在两个信号源间困惑（判断冲突）
- 占用 12.5% 提示词带宽（注意力稀释）
- 换装/换发型需同步修改 12 镜 txt（维护成本高）

### 7.2 第一性原理（公理化推导）

| 公理 | 含义 | 反例（v3.3 老格式） |
|---|---|---|
| 信息不重复 | 单一信息源不应被多个信号源重复承载 | 年龄/服装在档案段 + 参考图重复 |
| 注意力守恒 | AI 注意力带宽有限，重复信息占用核心信号带宽 | 档案段 300 字 = 12.5% 带宽 |
| 维护一致性 | 单一信息源更新成本最低 | 换装需同步修改 12 镜 txt |

### 7.8 v3.5 治本（M15·元注释剥离）

**Why**：v3.4 治本解决了"双信号源"（参考图 vs 文字描述），但残留了"元数据分离"问题——决策标注/修复说明/AI 友好说明/删除线标注 混入提示词主体，占复杂镜主体信号 18-34%（镜010+011 达 34%）。AI 推理困惑："哪些是'我要做什么'？哪些是'为什么这么做'？"

**核心原则**：元数据完全分离（公理 4）+ 信息密度均匀（公理 5）+ 自检工具化（公理 6）

#### 7.8.1 三公理（v3.5 新增）

- **公理 4（元数据分离）**：决策日志/修复说明/规则引用等元数据不应混入提示词主体
- **公理 5（信息密度均匀）**：每行主体内容应有 > 80% 新增价值
- **公理 6（自检工具化）**：自检段不应出现在提示词 txt 中，应由工具自动生成

#### 7.8.2 24 类元注释检测模式（M15_META_PATTERNS）

```python
M15_META_PATTERNS = [
    r'M11决策#?\d*', r'M9\s+Gap-?\d*', r'M12规则\d+',
    r'M13[^\s\u4e00-\u9fff]*', r'M14[^\s\u4e00-\u9fff]*', r'M15[^\s\u4e00-\u9fff]*',
    r'修正原版[^\s\u4e00-\u9fff]*', r'AI友好[^\s\u4e00-\u9fff]*',
    r'避免.{0,15}失败', r'避免.{0,15}双重',
    r'压缩原版\d*', r'🔴删[^\s\u4e00-\u9fff]*',
    r'凭空[^\s\u4e00-\u9fff]*', r'替代原版[^\s\u4e00-\u9fff]*',
    r'剧本未提[^\s\u4e00-\u9fff]*', r'9s长镜[^\s\u4e00-\u9fff]*',
    r'接近上限[^\s\u4e00-\u9fff]*', r'升华断点可接受[^\s\u4e00-\u9fff]*',
    r'场景突变[^\s\u4e00-\u9fff]*', r'\d+个微表情',
    r'微表情合并[^\s\u4e00-\u9fff]*', r'微表情压缩[^\s\u4e00-\u9fff]*',
    r'决策#\d+', r'Gap-\d+',
]
```

#### 7.8.3 关键约束（约束 24-26）

- **约束 24（元注释零容忍）**：提示词主体中不得出现决策标注（M11决策#XXX）、修复说明（**修正原版xxx** / **AI友好** / **避免双重失败** / **压缩原版** / **凭空细节**）、删除线说明等元注释。所有元数据应集中到决策日志或工具报告。
- **约束 25（自检段外置）**：每镜 txt 末尾不得包含【本镜自检】段。自检报告由 `validate_prompt.py v3.0` / `v35_selfcheck.py` 自动生成到外部报告。
- **约束 26（合规单点化）**：严禁字幕文字 等合规信息只保留一处（【合规】/【合规保留】行）。其他段落不得重复。

#### 7.8.4 v35_strip_meta.py 3 遍剥离

1. **第 1 遍（精确模式）**：META_PATTERNS 列表中的 60+ 精确正则
2. **第 2 遍（粗体段）**：META_BOLD_PATTERNS 检测 `**xxx**` 内含元关键词 → 整段剥离
3. **第 3 遍（括号段）**：paren_replacer 检测 `（xxx）` 内含元关键词 → 整段剥离

## 十、使用流程

```bash
# 1. 准备剧本（任意格式）
echo "剧本内容..." > input_script.txt

# 2. 调用 skill（伪代码，实际由 agent 执行）
python -m skill.generate \
  --input input_script.txt \
  --style auto \
  --duration 80 \
  --output skill-output/
```

### 完整使用（含手动配置）
```bash
# 1. 编辑 config.yaml
vim config.yaml
# 设置 style: 古早霸总
# 设置 duration: 80
# 设置 platform: 即梦

# 2. 调用 skill
python -m skill.generate --config config.yaml --input input_script.txt
```

USER: 我有一个古早霸总剧本，请生成分镜头提示词

  Step 4: 执行 M2（分镜设计）→ 输出到 00_总览/00_M2_分镜设计.md

## 十一、最佳实践

1. **先解析后设计**：不要跳过 M1 直接进入 M2
2. **风格预设优先**：先看 `styles/` 是否有匹配的，没有再考虑新建
3. **CUE 不要超过 3 个**：短视频注意力有限
4. **场景不要超过 4 个**：超过会拖慢节奏
5. **每镜 ≤1800 字（v3.7 治本）**：太少信息不足，太多 AI 抓不到重点
6. **剧本原词必保留**：对白 100% 不改
7. **图片资产替代文字描述**：【人物_XXX：】+ 图片参考，不在 prompt 中描述外貌
