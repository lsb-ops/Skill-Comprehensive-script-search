# M4 — 衔接校验模块

> 阶段4：校验所有镜的衔接一致性
> 输入：所有 txt 文件
> 输出：校验报告（issue 清单 + 修复建议）

---

## 一、模块目标

校验 M3 生成的所有提示词，确保：

1. **站位一致性**（180 度轴线）
2. **道具轨迹**（跨镜位置连贯）
3. **CUE 节奏**（时间点 + 效果合理）
4. **字数校验**（**≤1800 字 · v3.7 治本**）
5. **风格一致性**（色温/光比/视觉参考）
6. **物理一致性**（方向、速度、相对位置）

---

## 二、输入

- 所有 `镜XXX_xxx.txt` 文件
- 场景主控档案
- CUE 清单
- 色温曲线

## 四、校验维度

### 校验 1：站位一致性（180 度轴线）

**检查项**：
- 每镜人物在画面中的位置（左/中/右）
- 视线方向
- 多人物时是否遵守 180 度轴线
- 跨镜位置关系是否一致

**检查方法**：
```python
# 伪代码
for i in range(1, N):
    prev_shot = shots[i-1]
    curr_shot = shots[i]
    
    # 检查人物位置
    if prev_shot.person_A_position != curr_shot.person_A_position:
        if not is_axis_pivot(curr_shot):
            report_issue("跳轴", "严重")
```

**常见问题**：
- 双人对峙跳轴
- 单人方向变化
- 道具位置漂移

**修复策略**：
- 修正人物位置
- 插入轴线重建镜头
- 用浅景深掩盖

### 校验 2：道具轨迹

**检查项**：
- 道具位置跨镜是否连贯
- 道具状态是否一致（戴/收/丢）
- 道具延续关系是否标注

**检查方法**：
```python
# 伪代码
for prop in all_props:
    shots_with_prop = get_shots_with_prop(prop)
    for i in range(1, len(shots_with_prop)):
        prev = shots_with_prop[i-1]
        curr = shots_with_prop[i]
        
        # 检查状态连贯
        if prev.prop_state != curr.prop_state:
            if not has_transition(prev, curr):
                report_issue("道具状态跳变", "一般")
```

**常见问题**：
- 道具位置漂移（桌左 → 桌右）
- 道具状态跳变（戴 → 收，无过渡）
- 道具消失无说明

### 校验 3：CUE 节奏

**检查项**：
- CUE 数量（≤3）
- CUE 位置（开头/中段/结尾）
- CUE 之间间隔（≥25 秒）
- CUE 差异化（色温/光比/配乐）
- CUE 服务主题

**检查方法**：
```python
# 伪代码
cues = get_all_cues()
assert len(cues) <= 3, "CUE 数量超限"

for i in range(1, len(cues)):
    gap = cues[i].time - cues[i-1].time
    assert gap >= 25, f"CUE 间隔不足: {cues[i].time} - {cues[i-1].time} = {gap}秒"
```

**常见问题**：
- CUE 数量过多
- CUE 间隔太近
- CUE 无差异

### 校验 4：字数校验（v3.7 治本）

**检查项**：
- 总字数 **≤1800**（v3.7）
- 各段字数比例（按 1800 目标）
- 有台词镜含【声音_角色：】（v3.7 新增）

**检查方法**：
```python
# 伪代码（v3.7）
for shot in all_shots:
    char_count = count_chars(shot.content)
    assert char_count <= 1800, f"{shot.name} 字数超限: {char_count}"
    # v3.7 新增：声音引用校验
    if shot.has_dialogue() and "声音_" not in shot.content:
        warn(f"{shot.name} 有台词但缺声音引用")
```

**常见问题**：
- 字数过多（> 1800）
- 字数过少（< 1400）
- 段间字数失衡
- 有台词但缺声音引用（v3.7 新增）

### 校验 5：风格一致性

**检查项**：
- 色温曲线连贯
- 光比一致
- 色调一致
- 视觉参考一致
- 调色管线一致

**检查方法**：
```python
# 伪代码
for i in range(1, N):
    prev_shot = shots[i-1]
    curr_shot = shots[i]
    
    # 检查色温连贯
    if not is_temp_curve_valid(prev_shot.temp, curr_shot.temp):
        report_issue("色温跳变无说明", "一般")
    
    # 检查 LUT 一致
    if prev_shot.lut != curr_shot.lut and not has_transition(prev_shot, curr_shot):
        report_issue("LUT 变化无过渡", "一般")
```

**常见问题**：
- 色温跳变无说明
- LUT 风格不统一
- 光比突变

### 校验 6：物理一致性

**检查项**：
- 人物相对位置连贯
- 运动方向一致
- 速度连贯（不要突然加速/减速）
- 重力连贯（不要违反重力）

**检查方法**：
```python
# 伪代码
for i in range(1, N):
    prev_shot = shots[i-1]
    curr_shot = shots[i]
    
    # 检查运动方向
    if prev_shot.movement_direction != curr_shot.movement_direction:
        if not has_pivot(curr_shot):
            report_issue("运动方向跳变", "严重")
    
    # 检查速度连贯
    if prev_shot.speed != curr_shot.speed:
        if abs(prev_shot.speed - curr_shot.speed) > 1.0:  # 1 m/s 阈值
            report_issue("速度跳变", "一般")
```

**常见问题**：
- 人物突然反向
- 速度突变
- 违反重力（悬浮）

## 五、问题分类

### 严重（必须修复）
- 跳轴（180 度轴线违反）
- 物理不连贯（重力/方向违反）
- 字数严重超限（>3000 或 <1500）
- CUE 数量超限（>3）
- 人物外貌描述（违反资产化原则）

### 一般（建议修复）
- 字数轻微超限（>2400 或 <2000）
- 道具状态跳变无过渡
- 色温跳变无说明
- 风格参考过多（>4）

### 建议（可选优化）
- 动作可以更详细
- 光影可以更丰富
- 风格参考可以更精准
- 段落顺序可以更合理

## 七、自动化校验脚本

参考 `scripts/continuity_check.py`：

```python
#!/usr/bin/env python3
"""
M4 衔接校验自动化脚本
"""

import re
import os
import sys
from pathlib import Path

def count_chars(content):
    """计算字符数（中文）"""
    return len(content)

def check_axis(prev_shot, curr_shot):
    """检查轴线"""
    # 提取人物位置
    prev_positions = extract_positions(prev_shot)
    curr_positions = extract_positions(curr_shot)
    
    # 比较位置
    issues = []
    for person in prev_positions:
        if person in curr_positions:
            if prev_positions[person] != curr_positions[person]:
                if not is_pivot(curr_shot):
                    issues.append(f"{person} 位置跳变: {prev_positions[person]} → {curr_positions[person]}")
    return issues

def check_word_count(shot):
    """检查字数"""
    issues = []
    count = count_chars(shot['content'])
    if count < 2000:
        issues.append(f"字数过少: {count} < 2000")
    elif count > 2400:
        issues.append(f"字数过多: {count} > 2400")
    return issues

def check_cue_rhythm(shots):
    """检查 CUE 节奏"""
    cues = [s for s in shots if 'CUE' in s.get('title', '')]
    issues = []
    
    if len(cues) > 3:
        issues.append(f"CUE 数量超限: {len(cues)} > 3")
    
    # 检查 CUE 间隔
    for i in range(1, len(cues)):
        prev_time = cues[i-1]['time']
        curr_time = cues[i]['time']
        gap = curr_time - prev_time
        if gap < 25:
            issues.append(f"CUE 间隔不足: {cues[i-1].get('id')} → {cues[i].get('id')} ({gap} 秒)")
    
    return issues

def main():
    if len(sys.argv) < 2:
        print("Usage: continuity_check.py <shot_dir>")
        sys.exit(1)
    
    shot_dir = Path(sys.argv[1])
    shots = load_shots(shot_dir)
    
    issues = []
    for i, shot in enumerate(shots):
        # 字数检查
        issues.extend(check_word_count(shot))
        
        # 轴线检查
        if i > 0:
            issues.extend(check_axis(shots[i-1], shot))
    
    # CUE 检查
    issues.extend(check_cue_rhythm(shots))
    
    # 输出报告
    print(f"总问题数: {len(issues)}")
    for issue in issues:
        print(f"  - {issue}")

if __name__ == "__main__":
    main()
```
