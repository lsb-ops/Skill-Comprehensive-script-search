## 一、模块目标

把 M2 的分镜大纲转换为 AI 视频工具可直接使用的提示词，每个 ≤1800 字（v3.7 治本），套用 5 段式结构（KB01）。

---

## 二、输入

- M2 输出的分镜大纲
- M2 输出的场景主控档案
- M2 输出的衔接表
- M2 输出的 CUE 清单
- M2 输出的色温曲线

---

## 三、输出格式（v2.0 分层结构）

### 3.1 输出路径

**所有分镜 txt 统一写入：**
```
{output_root}/{剧本名}/01_分镜头/镜{NN}_{主题}.txt
```

**示例：**
```
生成验证/副本当霸道总裁拥有媒体素养_EP01/01_分镜头/
├── 镜001_张园蒙太奇登场.txt
├── 镜002_弄堂相撞慢动作.txt
└── ...
```

### 3.2 文件命名格式

`镜{NN}_{主题}.txt`
- **NN**：3 位数字（001-099）
- **主题**：4-8 字的简短主题（如"张园蒙太奇登场"）

### 3.3 前置动作：创建文件夹骨架

**M3 启动前，必须确认或创建以下文件夹结构：**

```python
# 伪代码
output_dir = f"{output_root}/{剧本名}"
os.makedirs(f"{output_dir}/00_总览",   exist_ok=True)  # M1/M2/M4 报告位
os.makedirs(f"{output_dir}/01_分镜头", exist_ok=True)  # 本模块写入位
os.makedirs(f"{output_dir}/02_场景主控", exist_ok=True)  # 空文件夹（用户图片槽）
os.makedirs(f"{output_dir}/03_人物档案", exist_ok=True)  # 空文件夹（用户图片槽）
os.makedirs(f"{output_dir}/04_道具档案", exist_ok=True)  # 空文件夹（用户图片槽）
```

**重要：02/03/04 只创建空文件夹，M3 不在其中写入任何文件。**

### 3.4 资产引用命名（与图片同名）

提示词的资产标记必须与用户图片文件名严格一致：

| 提示词标记 | 对应用户图片（用户手动放入） |
|---|---|
| `【人物_刘一鸣：】` | `03_人物档案/人物_刘一鸣.png` |
| `【道具_墨镜：】` | `04_道具档案/道具_墨镜.png` |
| `【场景_弄堂内：】` | `02_场景主控/场景_弄堂内.png` |

这样在即梦/可灵中可以直接 @ 同名图片。

### 3.5 文件内容结构（按 5 段式）

每个 txt 内部结构保持不变：

```
镜{NN} {主题} {时长}秒

[资产引用]
[人物资产档案（可选）]
[镜头参数]
[画面描述]
[光影氛围]
[技术参数]
[音效]
[BGM（可选）]
[台词]
[合规（可选）]
```

## 四、工作步骤

### 步骤 0：读取 4 源（M10 强制）

**未读取 4 源 → 拒绝进入步骤 1**

| 来源 | 必读项 | 用途 |
|---|---|---|
| 剧本原文 | .docx 解析后文本 | 台词 100% 还原 / 识别凭空补完 |
| M2 分镜设计 | 场景主控档案 + 衔接表 + CUE 清单 + 色温曲线 | 镜数规划 / 风格基调 |
| 上镜 TXT 末段 | 上镜的"画面描述"末段 | 站位 / 道具 / 机位继承 |
| 场景拓扑图 | 00_总览/00_场景拓扑图.md | 3 人物坐标 / 机位方向 |

**信息源**：场景主控档案 + 上镜衔接

**必备元素**：

### 步骤 3：填充段2：镜头参数

**信息源**：分镜大纲的镜头规格

**12 核心参数**（见 KB02）：
- 时长
- 画幅
- 景别
- 焦距
- 运镜
- 机位
- 高度
- 视角
- 视点
- 构图
- 景深

**示例**：


### 步骤 4：填充段3：画面描述

**信息源**：分镜大纲的动作 + 剧本原词 + 上镜衔接

**结构**：
- 按时序分段（0-X秒、X-Y秒、Y-Z秒）
- 每段描述：人物动作 + 表情 + 物理细节
- 保留剧本原词（标注"剧本原词"）

**示例**：
```
画面描述
0到3.5秒，CU近景85毫米长焦浅景深。资产人物小仇CU居中纵向，剧本原词瑟瑟发抖真实物理，肩膀加上臂加前臂的肌肉颤动真实物理v3.0强化每秒4到6次颤动频率。

```

### 步骤 5：填充段4：光影氛围

**信息源**：色温曲线 + CUE 清单 + 风格预设

**结构**：
- 主光 Key Light（位置 + 色温 + 强度）
- 辅光 Fill Light（位置 + 色温 + 强度）
- 轮廓光 Back Light（位置 + 色温 + 强度）
- 色温（具体 K 值）
- 光比
- 色调

**示例**：
```

主光Key Light，弄堂天光柔光4000K从画面左上方45度，强度90%延续镜009。辅光Fill Light，4500K冷蓝，强度30%延续镜009。轮廓光Back Light，5000K中性偏暖，强度80%延续镜009。色温4000K冷静科普美学延续镜009。光比1比2柔光。色调中性青橙加自然饱和。
```

**CUE 时刻**：

- 关键物理（本镜最核心的物理细节）
- 关键合规（字幕/文字禁忌）

**示例**：

### 步骤 9.5：不变量检查（M10 强制，8 项）

| # | 不变量 | 阈值 | 违规处理 |
|---|---|---|---|
| 1 | 物理细节数 | ≤ 8 个 / 镜 | 自动精简 |
| 2 | 手部微动作 | ≤ 3 个 / 镜 | 自动精简 |
| 3 | 文字锚点 | 0 处（"延续镜00X"） | 替换为参考图引用 |
| 4 | 场景空间坐标 | 段存在 | 自动补段 |
| 5 | 机位方向 | 已标注 | 自动补行 |

### 步骤 9.6：反幻觉 9 大规则自检（M12 强制）

| # | 规则 | 检查项 | 违规处理 |
|---|---|---|---|
| 1 | **单一动作原则** | 0-X 时段只描述 1 个核心动作（不叠加表情+手势+走位） | 拆分时段 |
| 2 | **物理量级强制** | 每个动作含 速度/角度/力度/频率 至少 1 个 | 自动补全 |
| 3 | **资产标记替代外貌** | 0 处外貌形容词（"俊美""高大"等） | 替换为【人物_XXX：】 |
| 4 | **场景空间坐标前置** | 段2 顶部必须含 3 人物坐标 + 机位方向 + 上镜→本镜位移 | 自动补段 |
| 5 | **风格参考 ≤ 3 部** | 参考电影 ≤ 3 部 | 删除多余 |
| 6 | **抽象词黑名单** | 0 处（蒙太奇美学/教化者美学/极具压迫感/戏剧光晕/AI渲染/电影级/8K/HDR/杜比视界） | 替换为物理描述 |
| 7 | **CUE 时刻物理跳变** | CUE 不写"渐变"而写"Xms 内从 XK 跳到 YK" | 改写 |
| 8 | **禁止反作用力/抛物线/帧率切换** | 0 处（"水珠反弹""镜内时间变慢""24→12fps"等） | 改写或拆分 |
| 9 | **同镜同景别** | 单镜内景别变化 ≤ 2 次（MS→CU 算 1 次跳变） | 拆分 |

**任一不通过 → 调整 → 回到步骤 1（最多 3 次）**

### 步骤 9.7：时长分级决策（M12 强制）

**失败率曲线**（基于 M12 报告实测数据）：
- ≤5s：5-8% 失败率（高成功）
- 6s：12% 失败率（标准）
- 7-8s：18-22% 失败率（需强约束）
- 9-10s：35-50% 失败率（强制拆分）
- 11-15s：55-70% 失败率（仅允许叠化合并）
- >15s：>75% 失败率（禁止）

**强制规则**：
| 时长 | 决策 | 例外 |
|---|---|---|
| **≤5s** | ✅ 直接使用 | 无 |
| **6s** | ✅ 标准单镜 | 无 |
| **7-8s** | ⚠️ 强约束（步骤 9.6 必过 + 简化物理细节到 ≤ 6 个） | 无 |
| **9-10s** | 🔴 **强制拆分**为两镜（每镜 ≤ 8s） | CUE 独立镜可保留 10s |
| **11-15s** | 🟡 **仅允许叠化合并**（场景 + 人物延续） | 单场景长镜可保留 |
| **>15s** | ⛔ **禁止生成**——必须拆分 | 无例外 |

**调用 6 角色机制**（M11 时机 4）：当 M2 给出 9-15s 候选时，**必须**触发 6 角色讨论是否拆分/合并。

**失败=0 原则**：成功率 < 70% 的方案禁止输出，宁可拆分/叠化也不冒险。

### 步骤 9.10：v3.5 治本·元注释剥离（M15 强制，3 项）

**核心原则**：元数据完全分离（公理 4）+ 信息密度均匀（公理 5）+ 自检工具化（公理 6）

**检测模式**（v35_strip_meta.py 实现 3 遍剥离）：

#### 第 1 项：元注释零容忍（约束 24）

**目标**：提示词主体中不得出现决策标注/修复说明/AI友好说明/删除线说明等元注释

**24 类元注释检测模式**（M15_META_PATTERNS）：
```python
M15_META_PATTERNS = [
    r'M11决策#?\d*', r'M9\s+Gap-?\d*', r'M12规则\d+',
    r'M13[^\s\u4e00-\u9fff]*', r'M14[^\s\u4e00-\u9fff]*', r'M15[^\s\u4e00-\u9fff]*',
    r'修正原版[^\s\u4e00-\u9fff]*', r'AI友好[^\s\u4e00-\u9fff]*',
    r'避免.{0,15}失败', r'避免.{0,15}双重',
    r'压缩原版\d*', r'🔴删[^\s\u4e00-\u9fff]*',
    r'凭空[^\s\u4e00-\u9fff]*', r'替代原版[^\s\u4e00-\u9fff]*',
    r'剧本未提[^\s\u4e00-\u9fff]*', r'9s长镜[^\s\u4e00-\u9fff]*',
    r'接近上限[^\s\u4e00-\u9fff]*', r'升华断点可接受[^\s\u4e00-\u9fff]*',
    r'场景突变[^\s\u4e00-\u9fff]*', r'\d+个微表情',
    r'微表情合并[^\s\u4e00-\u9fff]*', r'微表情压缩[^\s\u4e00-\u9fff]*',
    r'决策#\d+', r'Gap-\d+',
]
```

**3 遍剥离逻辑**：
1. **第 1 遍（精确模式）**：META_PATTERNS 60+ 精确正则
2. **第 2 遍（粗体段）**：`**xxx**` 内含 META_BOLD_PATTERNS → 整段剥离
3. **第 3 遍（括号段）**：`（xxx）` 内含元关键词 → 整段剥离

#### 第 2 项：自检段外置（约束 25）

**目标**：每镜 txt 末尾不得包含【本镜自检】段

**实施**：
- `strip_self_check_section()` 自动剥离 `【本镜自检·M9+M12+M13+M14】` 整段
- 自检报告由 `validate_prompt.py v3.0` / `v35_selfcheck.py` 自动生成到外部报告

#### 第 3 项：合规信息单点化（约束 26）

**目标**：严禁字幕文字 等合规信息只保留一处（【合规】/【合规保留】行）

**实施**：

### 同场景合并

**关键**：同场景多镜合并为一次生成（AI 视频工具特性）。

| 同场景镜号 | 合并策略 |
|---|---|
| 镜 003 + 镜 004 + 镜 005 | 一次性生成 25 秒，分镜剪辑 |
| 镜 009 + 镜 010 | 一次性生成 15 秒 |

**优势**：
- 人物一致性更好
- 场景一致性更好
- 节省生成时间
- 后期剪辑更顺畅
