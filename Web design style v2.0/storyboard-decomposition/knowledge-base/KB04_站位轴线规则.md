# KB04 — 180度轴线规则（Screen Direction / 180-Degree Rule）

> 防止"跳轴"的最重要摄影规则
> 多机位对话 / 追逐 / 对峙场景的核心约束

---

## 一、什么是轴线

### 定义
**轴线（Action Axis / Imaginary Line）**是连接两个主体（人物、物体、运动方向）的假想直线。

### 核心规则
**所有机位必须保持在轴线的同一侧**（180度半球内）。

如果跨越轴线，观众会产生**空间迷失感**——人物 A 突然"换了方向"，人物 B 突然"反了位置"。

---

## 二、轴线类型

### 类型1：人物轴线
两个角色对话时，连接两人的直线就是轴线。

```
A ←─轴线─→ B

机位1（左侧）  机位2（右侧）
都在轴线下方 → 合规
```

### 类型2：运动轴线
人物或物体沿某方向运动，运动方向的平行线就是轴线。

```
人物走向 →

机位在运动方向左侧 → 合规
```

### 类型3：视线轴线
人物看某个方向，视线的延伸线就是轴线。

```
人物视线 → 物体

机位在视线两侧 → 合规
```

---

## 三、为什么需要轴线规则

### 反例：跳轴

```
场景：男女对话
镜1：A 在左，B 在右（建立）
镜2：突然跳到轴线另一侧
    → A 突然在右，B 突然在左
    → 观众空间迷失
    → 剪辑灾难
```

### 后果
- 观众产生**方向困惑**
- 破坏**空间连续性**
- AI 生成多镜无法衔接
- 后期剪辑困难

## 四、合规示例

### 4机位对话场景
```
         A ←─轴线─→ B
          ↓        ↓
        机位1   机位2
        机位3   机位4

所有机位都在轴线下方（180度半球内）
```

```
镜1（MS双人中景）：机位1（轴线下方居中偏左）
    A 在左三分，B 在右三分，A 视线朝右，B 视线朝左

镜2（CU A 特写）：机位3（轴线下方偏左，A 同侧）
    A 在画面中心偏右，视线朝画右（看着 B 的方向）

镜3（CU B 特写）：机位4（轴线下方偏右，B 同侧）
    B 在画面中心偏左，视线朝画左（看着 A 的方向）

镜4（MS 双人中景）：机位2（轴线下方居中偏右）
    A 在左三分，B 在右三分（与镜1 一致）
```

**关键**：所有镜头 A 永远在左或右的同一侧。

## 五、轴线越界场景的处理

### 场景1：角色走动越过轴线
**处理**：插入**轴线重建镜头**（如 MS 重新建立空间关系）。

```
镜1：A 在左，B 在右，A 走向右
镜2：A 走到右，B 在左（自然越线）
镜3：插入 MS 双人中景，从新角度重建（A 在左，B 在右 重新建立）
镜4：A 在左 CU，B 在右 CU（继续）
```

### 场景2：摄影机需要越线
**处理**：插入**运动轴线镜头**（摄影机本身越过轴线）。

```
镜1：机位1 拍摄 A 在左
镜2：摄影机从左向右移动 180 度（Dolly）
镜3：机位2 拍摄 A 在右（已越线，但摄影机移动被观众感知）
```

### 场景3：戏剧性反打（故意越线）
**处理**：用**特写 + 浅景深**让观众失去空间参考。

```
镜1：A 在左中景（建立）
镜2：A CU 极浅景深（背景虚化，参考丢失）
镜3：A 在右 MS（中景建立新关系，观众接受）
```

## 六、短剧常见机位布局

### 对话场景（2人）

```
        A ←─轴线─→ B
         ↓        ↓
       居中     居中

镜1：MS 居中（建立）
    A 在画面左三分之一，视线朝右
    B 在画面右三分之一，视线朝左

镜2：CU A（机位在 A 同侧）
    A 在画面中心偏右，视线朝画右
    背景虚化（B 在焦点外）

镜3：CU B（机位在 B 同侧）
    B 在画面中心偏左，视线朝画左
    背景虚化（A 在焦点外）
```

### 对峙场景（2人面对面）

```
         A
         ↑
       轴线
         ↓
         B

机位在轴线两侧（斜后方）
镜1：从 A 背后拍 B（B 在画面中心，A 背影模糊前景）
镜2：从 B 背后拍 A（A 在画面中心，B 背影模糊前景）
镜3：侧面 MS（轴线下方居中）
```

### 走路场景（单方向运动）

```
人物走向 → 前进方向

机位在人物侧前方（与运动方向呈 45-90 度）
机位在人物背后（POV 跟随）
机位在人物正侧（横移平行）
```

## 七、AI 视频生成中的轴线规则

### 问题
AI 生成多镜视频时，最常见的失败是**人物朝向不一致**。

镜1 人物朝左走，镜2 突然朝右走，观众立刻察觉。

### 解决方案

#### 方案1：建立镜明确标注朝向
```
资产人物刘一鸣，MS 居中，视线朝画右，面对小仇。
```

#### 方案2：每镜标注站位
```
0到3秒，资产人物刘一鸣在画面左侧三分之一，资产人物小仇在画面右侧三分之一。
```

#### 方案3：使用轴线锚点
```
【轴线锚点-001】所有涉及刘一鸣/小仇的镜头，A 在画面左侧，B 在画面右侧。
```

## 八、轴线规则的常见错误

### 错误1：双人对话随意跳轴
```
镜1：A 左 B 右 ✓
镜2：A 右 B 左 ✗（跳轴）
```
观众立刻发现"他们位置换了"。

### 错误2：单人镜头朝向不一致
```
镜1：人物朝左走
镜2：人物朝右走
镜3：回到朝左走
```
观众迷失。

### 错误3：道具位置漂移
```
镜1：道具在桌上左边
镜2：道具突然在桌上右边
```

### 错误4：环境元素位置变化
```
镜1：窗在画左
镜2：窗在画右
```

## 九、轴线规则在 AI 提示词中的表达

### 必备元素

每镜应明确：
1. **人物/物体在画面中的位置**（左/中/右，三分/中心）
2. **人物视线方向**（朝画左/画右/画面内）
3. **运动方向**（前进/后退/横移方向）
4. **道具相对位置**（与人物的空间关系）

### 示例（合规）

```
0到3秒：MS 中景，居中构图。
  资产人物刘一鸣在画面左侧三分之一，视线朝画右，看向小仇。
  资产人物小仇在画面右侧三分之一，视线朝画左，看向刘一鸣。
  两人之间是 180 度轴线，所有后续镜头保持此关系。

3到5秒：CU 刘一鸣特写。
  刘一鸣在画面中心偏右，视线朝画右（轴线右侧）。
  浅景深，背景小仇虚化。

5到7秒：CU 小仇特写。
  小仇在画面中心偏左，视线朝画左（轴线左侧）。
  浅景深，背景刘一鸣虚化。
```

## 十、轴线规则的"破例"技巧

### 破例1：主观视角（POV）
观众从某角色视角看世界，方向自然变化。

```
镜1：A 看向画右的 B（建立 A 在左 B 在右）
镜2：POV 从 A 的视角看 B（B 在画右，A 视线延续）
```

### 破例2：旋转镜头
摄影机绕人物旋转 180 度，但摄影机本身在动，观众能感知。

### 破例3：剪影/极度浅景深
当景深极浅（f/1.4），背景完全虚化，观众失去空间参考。

### 破例4：插入轴线重建镜头
跨轴前插入一个 MS 中景镜头，让观众"重置"空间。

- [ ] 道具/环境元素位置是否连贯？
- [ ] 跳轴前是否插入重建镜头？

## 十二、短剧常用轴线配方

### 配方1：双人对峙
```
轴线：连接两人的直线
机位：轴线下方 45 度斜后方
镜1：MS 居中（建立）
镜2-3：CU 正反打
镜4：MS 居中（收束）
```

### 配方2：追逐
```
轴线：与运动方向平行
机位：与运动方向呈 30-90 度
镜1：LS 远景（建立）
镜2：MS 跟拍（侧面）
镜3：CU 表情（侧面）
镜4：MS 拉开（结束）
```

### 配方3：单人独白
```
轴线：视线方向
机位：视线方向后方 45 度
镜1：CU 居中（建立）
镜2：MCU 居中（拉远）
镜3：CU 居中（推回）
```

### 配方4：群戏（3-5人）
```
轴线：组中心为虚拟轴
机位：环绕轴线
镜1：MS 群像（建立）
镜2-4：单人 CU（各自同侧）
镜5：MS 群像（收束）
```

## 十三、v3.7 治本·跨镜站位继承约束（**强制**）

> **Why**：26 镜 EP01 实测发现，6 类站位问题反复出现（11 镜 v3.7 初版 4 处中重度问题）。
> v3.5 治本（去元注释）+ v3.7 治本（≤1800 + 声音参考）解决了字面问题，
> 但**跨镜站位继承**仍是 AI 生成最大盲区。本节强制约束跨镜站位继承。

### 13.1 站位必须明确（**强制**）

**每镜画面描述段必须包含以下元素**：

```
0-X 秒，[景别]+[焦距]。[机位描述]。
资产人物[角色名]在画面[左/中/右/前/后][位置描述]，[动作]...
```

#### ❌ 反例（v3.7 治本前）
```
资产人物刘一鸣CU前景模糊+西装袖口被泼湿真实布料物理。
```
**问题**：没说明在骆骆左侧还是右侧，AI 生成时可能放错位置

#### ✅ 正例（v3.7 治本）
```
资产人物刘一鸣在骆骆左侧CU前景模糊+西装袖口被泼湿真实布料物理（明确左侧承接镜003）。
```
**修正**：必须标注与其他角色的相对位置

### 13.2 跨镜站位继承（**强制**）

**每镜开头必须明示"承接上镜"或"重建站位"**：

| 状态 | 标注格式 | 示例 |
|---|---|---|
| 继承上镜 | `延续镜00X` | `资产人物骆骆MS居中偏右（延续镜002）` |
| 位置重建 | `（X秒位置重建）` | `资产人物刘一鸣MS居中（CU切MS位置重建）` |
| 离场承接 | `承接镜00X离场` | `画面外已离去（承接镜003后退离场）` |
| 入画承接 | `接镜00X入画` | `资产人物小仇前景CU偏左入画（接镜006小仇呆滞延续）` |

### 13.3 道具状态跨镜延续（**强制**）

**关键道具状态必须跨镜标注**：

```
【道具_墨镜：拿在手里左手（延续镜004摘下→镜005手持→镜008戴回→镜010收内袋）】
【道具_名片：骆骆手持逃离关键道具（镜007接过→镜007逃离后消失）】
【道具_眼镜：银框金属圆形细框（镜003小仇戴→镜006仍戴→镜008仍戴→镜010仍戴）】
```

**Why**：11 镜实测发现墨镜状态 3 镜跳变（007 拿在手里 → 008 不明 → 009 戴在脸上），
道具状态跳变是 AI 生成最常见的物理不一致问题。

### 13.4 角色离场/入画承接（**强制**）

#### 离场（角色从画面内 → 画面外）

**必须标注承接上镜的离场动作**：

```
【人物_骆骆：画面外已离去（承接镜003骆骆后退半步+转身离场）】
【人物_刘一鸣：画面外已离去（承接镜005转身走开1.5m/s）】
```

#### 入画（角色从画面外 → 画面内）

**必须标注承接上镜或新出场**：

```
【人物_小仇：瞬间闪现（接镜002骆骆相撞→镜003小仇背景右侧入画）】
【人物_小仇：前景MCU偏右入画（接镜005主讲延续到镜006反应）】
```

### 13.5 站位继承 5 个模板

#### 模板1：同景别跨镜（继承）
```
镜005 MS居中 → 镜006 MS居中
资产人物刘一鸣MS居中纵向（延续镜005）
```

#### 模板2：景别后退（位置重建）
```
镜005 CU居中 → 镜006 MS居中
资产人物刘一鸣MS居中纵向（CU切MS位置重建，承接镜005位置）
```

#### 模板3：景别推近（位置继承+聚焦）
```
镜005 MS居中 → 镜006 MCU居中
资产人物刘一鸣MCU居中纵向（MS切MCU聚焦继承，承接镜005居中位置）
```

#### 模板4：主角切换（前景/后景跳变）
```
镜005 刘一鸣主讲 → 镜006 小仇主讲
资产人物小仇CU居中纵向（接镜005主讲延续，刘一鸣前景模糊出画）
```

#### 模板5：角色离场（画面外承接）
```
镜005 主讲 → 镜006 离场
【人物_刘一鸣：画面外已离去（承接镜005转身走开1.5m/s）】
```

### 13.6 跳轴检测清单

**每镜生成后检查**：

- [ ] 同角色跨镜位置是否一致（左↔左 / 右↔右 / 中↔中）
- [ ] 双人场景两人相对位置是否保持（A左B右 → 下一镜仍A左B右）
- [ ] 道具位置是否跨镜延续
- [ ] 角色离场是否有承接说明
- [ ] 角色入画是否有承接说明
- [ ] 站位跳变是否标注"位置重建"
- [ ] 画面外角色是否有承接上镜说明
