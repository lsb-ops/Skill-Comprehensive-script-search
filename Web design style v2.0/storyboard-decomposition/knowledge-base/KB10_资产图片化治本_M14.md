## 一、问题本质（第一性原理）

### 1.1 用户洞察

> "由于我已经给了参考图片，关于人物的描述已经不重要了，这个是需要进行优化...并在 skill 里面进行加入相关的，这是非常关键的"

### 1.2 双信号源冲突模型

```
信号 A（提示词文字）："30岁男性。全黑西装+深蓝衬衫+Prada黑色领带+黑色短发自然垂坠"
        ↓
信号 B（参考图）：03_人物档案/人物_刘一鸣.png
        ↓
   AI 推理：两个信号源承载同一信息
        ↓
   三个可能结果：
   1) 不一致 → 困惑（判断冲突）
   2) 冗余 → 占用 12.5% 提示词带宽（注意力稀释）
   3) 矛盾 → 生成结果不可控（信号冲突）
```

### 1.3 公理化推导

**公理 1（信息不重复）**：单一信息源不应被多个信号源重复承载
- 反例：年龄/服装/发型/表情 既在参考图又在档案段 → 违反公理 1

**公理 2（注意力守恒）**：AI 注意力带宽有限，重复信息占用核心信号带宽
- 单镜提示词 2000-2400 字（v3.7 后已升级到 ≤1800 字）
- 档案段平均 300 字 = 12.5% 带宽
- 这 12.5% 用于"参考图已承载信息" → 0% 新增价值

**公理 3（维护一致性）**：单一信息源更新成本最低
- 现状：换装需同步修改 12 镜 txt
- 应为：换装只需更新 03_人物档案/图片

### 1.4 5 Whys（追溯到根因）

1. **Why 1**：为什么档案段要描述外貌？
   → 因为担心 AI 不看参考图 → 但 AI 同时接收文字 + 图片，参考图本身已足够

2. **Why 2**：为什么担心 AI 不看参考图？
   → 因为 v3.3 之前的 txt 没有强制使用"参考图引用"段（v3.2 才新增）

3. **Why 3**：为什么 v3.3 之后还保留文字描述？
   → 因为 T1 模板只更新了"参考图引用"段，未触及"人物资产档案"段

4. **Why 4**：为什么 T1 模板未触及档案段？
   → 因为档案段被设计为"图片路径说明"（v2.5 时代），未意识到参考图系统已替代此功能

5. **Why 5（根因）**：模板/工具链/资产引用 三者未同步升级，导致档案段从"路径说明"过时为"重复描述"却无人察觉。

## 二、v3.4 治本方案

### 2.1 核心原则

**参考图已承载信息则提示词不重复**

### 2.2 实施方式

#### 原 v3.3 格式（❌ 重复描述）

```markdown
人物资产档案
刘一鸣：30岁男性。全黑西装+深蓝衬衫+Prada黑色领带+黑色短发自然垂坠。戴飞行员墨镜金属框（0到5秒遮挡面部）。表情面无表情霸总冷漠。
小仇：28岁男性（比刘一鸣矮半头）。银框金属眼镜圆形细框（冷蓝色镜面反光）+浅蓝色西装外套+白色印花T恤。表情极度夸张戏精本精。
场景_张园外景：上海张园石库门建筑群+青砖墙+复古路灯+藤蔓+5000K自然日光（上午10点直射日光）。严禁字幕文字：魔新传媒集团/总裁/纳斯达克/全球传媒命脉。
```

#### 新 v3.4 格式（✅ 参考图指针）

```markdown
人物资产档案（v3.4 治本·参考图已承载信息则提示词不重复）
【本镜人物外貌/服装/发型/表情 全部由参考图承载，提示词不重复描述】
【参考_人物_刘一鸣：03_人物档案/人物_刘一鸣.png（关键特征·参考图承载）】
【参考_人物_小仇：03_人物档案/人物_小仇.png（关键特征·参考图承载）】
【参考_场景_张园外景：02_场景主控/场景_张园外景.png（关键特征·参考图承载）】
【合规保留】严禁字幕文字：魔新传媒集团/总裁/纳斯达克/全球传媒命脉
```

### 2.3 保留项 vs 删除项

| 类别 | 保留 ✅ | 删除 ❌ |
|---|---|---|
| **人物** | 合规状态（如"无墨镜·露脸"） | 年龄/性别/身高/服装/发型/表情 |
| **道具** | 合规/禁忌信息 | 颜色/材质/形状/品牌 |
| **场景** | 合规信息（如"严禁字幕文字"） | 建筑/装饰/天气/光影 |

### 2.4 关键约束

**约束 23（M14 强制·v3.4 治本）**：
> 人物资产档案段不得重复描述参考图已承载信息（年龄/身高/服装/发型/表情/道具外观/场景细节），全部由参考图承载；仅可保留合规/禁忌等图片无法承载的信息。

## 三、工具链实现（M14）

### 3.1 validate_prompt.py v2.0 → v2.1

**新增第 7 项自检**：`check_asset_archive_no_redundancy()`

**检测模式**：
```python
M14_REDUNDANT_PATTERNS = {
    "年龄": r"\d{1,2}岁(男|女)性",
    "身高": r"1\d{2}cm",
    "服装颜色+品类": r"(全黑|全白|深蓝|浅蓝|浅蓝色|黑色|白色|红色|蓝色|绿色|银框|金色|棕色)(西装|衬衫|T恤|领带|外套|眼镜|短袖|皮鞋|墨镜|外套|夹克|连衣裙|裤子)",
    "发型描述": r"(黑色|棕色|金色|银白)(短发|长发|卷发|直发|寸头|马尾|披肩发|自然垂坠|自然)(垂坠|发型|短发|长发)?",
    "表情描述": r"表情(极度|非常|十分)?(夸张|冷漠|戏精|严肃|温柔|冷酷|无)",
    "道具外观": r"(锃亮|光亮|哑光|磨砂)(黑色|白色|银色|金色)(皮鞋|墨镜|眼镜|手表|手机|钱包|包|领带)",
    "场景细节": r"(上海|北京|广州|深圳|杭州|成都)(张园|石库门|弄堂|外滩|老街|胡同)(建筑群|青砖墙|石板路|复古路灯|藤蔓)",
}
```

**豁免机制**：
- v3.4 治本格式（段首有"v3.4 治本"标注）只允许"（关键特征·参考图承载）"括号注释
- 若括号前内容 ≤ 30 字符 → 合规
- 否则 → 报告为"未压缩描述"

### 3.2 v34_refactor_archive.py v1.0 NEW

**核心逻辑**：
1. 读取文件 → 提取"人物资产档案"段
2. 解析实体（人物/道具/场景）by 已知资产名
3. 生成 v3.4 格式（1 行声明 + N 行参考图指针）
4. 保留"严禁字幕文字"等合规信息
5. 写回文件

**关键设计**：
- 项目级资产名清单（KNOWN_CHARACTERS / KNOWN_PROPS）
- 合规关键词白名单（COMPLIANCE_KEYWORDS）
- 资产版本管理（防止跨镜参考图不一致）

## 六、失败案例分析（预防）

### 6.1 失败模式 1：保留描述未识别

**场景**：自动重构时未识别"无名实体"（不在 KNOWN_CHARACTERS/PROPS 列表）

**预防**：
- 资产名清单需扩展为项目级元数据
- 重构脚本输出"未识别实体"清单供人工 review

### 6.2 失败模式 2：合规信息丢失

**场景**：重构脚本误将"严禁字幕文字：xxx"识别为角色描述并删除

**预防**：
- COMPLIANCE_KEYWORDS 白名单（严禁/禁止/合规）
- 这些行单独输出为 `【合规保留】xxx` 段
- 重构后必须人工 review 合规段

### 6.3 失败模式 3：跨镜参考图不一致

**场景**：镜003 引用 `人物_小仇.png`，镜005 引用 `人物_小仇_v2.png`

**预防**：
- 资产命名规范必须在项目早期确定
- M11 决策记录中明确"资产版本"字段

## 七、关键修复模式（推广到 KB 全部约束）

### 7.1 v3.4 治本的 3 个关键修复模式

1. **self-check section strip**（v3.3 沿用）
   - 问题：txt 末尾"本镜自检"段是元注释，被误判为违规
   - 修复：所有 v2.x 工具都剥离 `【本镜自检` 之前的部分

2. **画幅/光比正则升级**（v3.3 沿用）
   - 问题：v1.0 正则要求"9比16"格式，v3.3 txt 用"9:16"格式
   - 修复：正则改为 `画幅\s*\d+[比:：]\d+` 同时接受两种格式

3. **v3.4 治本·档案段去重**（NEW）
   - 问题：档案段文字描述与参考图重复，AI 在两个信号源间困惑
   - 修复：档案段改为"1 行声明 + N 行参考图指针"，检测模式 M14_REDUNDANT_PATTERNS
