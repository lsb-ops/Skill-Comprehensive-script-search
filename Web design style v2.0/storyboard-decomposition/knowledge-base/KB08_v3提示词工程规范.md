# KB08 — v3 提示词工程规范（2026-06-17）

> **强制约束**：M3 生成的所有单镜提示词必须遵守本规范。
> 本规范基于对国内 11 个分镜 txt 的逐字审阅（v3.0强化 323 次过载 / 真实物理 347 次过载 / T-stop 0 次缺失 / 抽象词 30+ 次冗余）。
> 目标：让 AI 视频生成（可灵/即梦/Sora/Runway）输出**真人感 + 物理正确 + 电影感**，去掉 AI 感。

---

## 一、6 条强制约束（不可违反）

### 约束 1：`v3.0强化` 单镜出现 ≤ 2 次

| 错误示例（实际：26-37 次/镜） | 正确示例（≤ 2 次/镜） |
|---|---|
| 每段每句都加 `v3.0强化` | 标题段 + CUE 段各 1 次（其他全删） |

**原因**：T5-XXL / CLIP-L 文本编码器 token 数 77-256 上限，高频词被 AI 当**无意义 token**忽略，失去信号作用。

### 约束 2：`真实物理` 替换为具体物理数字

| 错误示例 | 正确示例 |
|---|---|
| `真实物理`（抽象） | `G=-9.8 m/s²` / `角动量滞后 0.3s` / `Hertz 接触 0.02s` |
| `真实镜头物理` | `Dolly In 0.3米每秒` / `Dolly Out 0.3米每秒` |
| `真实物理行走速度` | `1.5米每秒真实行人速度` |
| `真实物理反作用力` | `反作用力 0.2米推后真实物理` |

**原因**：AI 不识别"真实物理"作为信号，但能识别**具体数字+单位**作为权重 token。

**重要保留词**：`真实物理引擎渲染`（即梦关键信号词）每镜 1-2 次保留。

### 约束 3：每镜必加 T-stop 值

| 场景 | T 值 + 焦距组合 |
|---|---|
| 远景/全景（LS） | `T8 + 24mm 深景全幅` |
| 标准中景（MS） | `T2.8 + 35mm 浅景深焦外二线性` |
| 中近景（MCU） | `T2 + 50mm 浅景深` |
| 近景/特写（CU） | `T1.4 + 85mm 极浅景深` |
| 极特写（ECU） | `T1.4 + 135mm 极浅景深+强压缩` |

**原因**：T-stop 是景深物理的**唯一**信号锚点，缺失时 AI 默认深景深，背景清晰 = 视频感重。

### 约束 4：抽象词黑名单（每镜 ≤ 1 次）

| 禁用词 | 原因 |
|---|---|
| `蒙太奇美学` / `教化者美学` / `冷静科普美学` | 抽象标签，AI 自由发挥导致混乱 |
| `极具压迫感` / `戏剧光晕` / `蒙太奇霸气美学` | 抽象感觉描述，AI 无法实现 |
| `美轮美奂` / `震撼` / `极致` / `唯美` | 完全无信号量 |
| `AI 后期逆光晕` | "AI 后期"是反信号，会触发 AI 自检 |
| `AI 后期` / `AI 渲染` / `AI 增强` | 元注释无意义 |
| `8K` / `HDR` / `杜比视界` / `电影级` | KB07 已禁 — 堆砌稀释主语 |

**例外保留**：
- `升华美学`（CUE③ 升华镜可用 1 次）
- `真实物理引擎渲染`（即梦 hot-token，每镜 1-2 次保留）

### 约束 5：色温与时间/天气匹配

| 场景 | 时间/天气 | 色温 | 光比 |
|---|---|---|---|
| 日景晴天 | 上午10点直射 | **5000-5500K** | 1:2.5 |
| 日景阴天 | 阴天漫射 | **6500-7000K** | 1:1.5 |
| 黄昏/日落前 | Golden Hour | **3000-3500K** | 1:3 |
| 夜景暖内 | 室内钨丝 | **3200K** | 1:2 |
| 夜景冷街 | 街灯 | **4500-5000K** | 1:3 |
| 蓝调时刻 | 日落后20min | **8000-10000K** | 1:4 |
| 冷静科普（CUE②） | 室内/阴天 | **4000K** | 1:2-1:3 |
| 升华（CUE③） | 逆光 | **5500K** | 1:∞ |

**错误**：日景外景写 4500K → AI 误读为黄昏。
**正确**：日景外景统一 5000K+。

### 约束 6：风格参考 ≤ 3 部

| 错误示例（4-5 部超载） | 正确示例（2-3 部最强） |
|---|---|
| 教父+花样年华+低俗小说+大话西游+功夫 | 教父+花样年华 |
| 功夫+花样年华+教父+低俗小说 | 功夫+教父 |

**格式**：`风格参考：X；副参考：Y`（用分号分隔，避免 "+" 串扰）

## 二、信号锚点 vs 噪音原则

### 2.1 信号锚点（高密度关键词）

**单镜内信号锚点必须包含**（按优先级）：
1. **物理数字**：G=-9.8 m/s² / 0.3s 滞后 / 1.5米每秒 / 45°
2. **光学指纹**：T2.8 / 35mm / 焦外二线性 smooth bokeh
3. **质感锚点**：皮肤微血管红晕 + 自然毛孔 + 发根阴影 + 发丝单丝级非块状
4. **物理交互**：Hertz 接触 0.02s / 角动量滞后 / G=9.8 重力
5. **品牌锚点**：ARRI Alexa LF + Signature Prime 镜头 / Cooke S4/i 焦外漩涡
6. **风格参考**：2-3 部最强电影名

### 2.2 噪音（必须删除）

| 噪音类型 | 表现 | 修复 |
|---|---|---|
| 抽象词 | `蒙太奇美学` 重复 5 次 | 删 80% |
| 真实物理 | `真实物理` 重复 30 次 | 替换为具体数字 |
| v3.0强化 | 重复 30 次 | 保留 1-2 处 |
| 元注释 | `AI画面严禁` / `AI渲染` | 每镜 1 处即可 |
| 价值形容词 | `精致` / `完美` / `无瑕` | 全部删除 |

## 九、v3.0 强化 使用规则

`v3.0强化` 是国内即梦的语义 hot-token，KB06 记录为重要信号。

**正确用法**：
- 单镜出现 ≤ 2 次
- 位置：标题段 + CUE 段（或关键技术参数段）
- 含义：标记"本镜进入 v3.0 强化风格"
- 作用：让 AI 进入高密度电影感渲染模式

**错误用法**：
- 每段每句都加（信号过载 = 噪音）
- 与 `真实物理引擎渲染` 重复堆叠
- 在 Sora/Runway 中使用（这些模型不识别 v3.0 信号）

**与其他信号词关系**：
- 即梦：`v3.0强化` + `真实物理引擎渲染` 双 hot-token 效果最强
- 可灵：`物理正确` + `真实速度` + `动态连贯` 敏感
- Sora：`cinematic` + `photorealistic` + `practical lighting` 敏感
- Runway：`hyperrealistic` + `8K` + `no artifacts` 敏感
- **不要跨平台混用 hot-token**
