# README

## 通用分镜头提示词生成 Skill

> 从任意剧本生成 ≤1800 字（v3.7 治本）的 AI 视频分镜头提示词
> **v2.6 — v3.3 重构闭环 + v3.4 治本（M14 资产档案去重）+ v3.7 治本（字数 ≤1800 + 声音参考沉淀）**（AI 一次成功率 5-15% → 80-95%）

---

## 快速开始

### 输入
- 任意剧本（小说/故事大纲/分场剧本/.docx 文件）

### 输出（v2.0 分层结构 + v3.3 重构版）
```
{output_root}/
└── {剧本名}/                          # 自动提取剧本名
    ├── 00_总览/                      # M1-M13 元数据文档
    ├── 01_分镜头/                    # N 个分镜 txt（核心 AI 输入）
    │   └── v3.3_重构版/              # v3.3 重构 txt（按时长分级 + 关键修复）
    ├── 02_场景主控/                  # 空文件夹（用户手动放场景参考图）
    ├── 03_人物档案/                  # 空文件夹（用户手动放人物参考图）
    └── 04_道具档案/                  # 空文件夹（用户手动放道具参考图）
```

**职责分工：**
- ✅ skill 自动生成：`00_总览/` + `01_分镜头/`（含 v3.3_重构版/）
- ✅ skill 创建空文件夹：`02_场景主控/` + `03_人物档案/` + `04_道具档案/`
- 👤 用户手动填充：在 `02/03/04` 中放入与提示词标记同名的 `.png` 图片

### 核心特点
1. **通用性强**：支持 8 种风格预设（古早霸总/都市情感/古装江湖/科幻未来/校园青春/悬疑推理/战争军事/家庭伦理）
2. **结构化输出**：5 段式结构 + 6 字段场景空间坐标段 + 本镜自检段
3. **字数精准**：每镜 ≤1800 字（v3.7 治本，按时长分级调整）
4. **风格一致**：通过 CUE 设计 + 色温曲线 + 视觉参考库保证
5. **衔接友好**：每镜标注上镜结束位置+本镜 AI 生成策略
6. **v3.3 重构闭环**：物理细节 356→≤96 / 文字锚点 60+→0 / 凭空补全 18→0
7. **v3.4 治本**：人物资产档案段去重（参考图已承载信息则提示词不重复），档案段字数 -18%
8. **M11 决策透明**：6 角色投票机制，5 个决策 #001-#005 完整记录

## 使用方法

### 方法 1：AI Agent 调用
直接将剧本和"使用 skill 生成提示词"的指令发给 AI Agent，Agent 会自动执行 4 阶段工作流。

### 方法 2：人工执行
1. 阅读 `SKILL.md` 了解工作流
2. 按顺序执行 M1 → M2 → M3 → M4
3. 每个阶段参考对应的 KB 文件
4. 加载对应的 STYLE 预设
5. 套用 T1 模板生成
6. **v3.3 重构时**：参考 KB09 + M13 报告

### 方法 3：脚本调用
```bash
# 基础校验
python scripts/detect_style.py input_script.txt
python scripts/validate_prompt.py output/镜001.txt

# v3.3 + v3.4 批量校验（M13 6 项 + M14 第 7 项自检）
python scripts/validate_prompt.py output/v3.3_重构版/ --batch

# 跨镜衔接校验（M4 v3.3 增强）
python scripts/continuity_check.py output/

# v3.4 治本·批量重构人物资产档案段（解决"提示词 vs 参考图"双信号冲突）
python scripts/v34_refactor_archive.py output/v3.3_重构版/ --batch
```

## 核心设计理念

### 1. 通用 vs 风格分离
- **通用知识**：5 段式结构、摄影语言、站位规则、CUE 概念 → 100% 通用
- **风格预设**：色温数值、视觉参考、典型结构、标配元素 → 通过 STYLE 配置

### 2. 衔接友好
- 每镜标注"上镜结束位置"
- 每镜标注"本镜 AI 生成策略"
- 建议同场景多镜合并为连续生成段

### 3. 资产图片化
- 提示词中不描述人物外貌
- 通过【人物_XXX：】+ 图片参考实现一致性
- 用户自行管理图片资产库

### 4. 剧本原词 100% 保留
- 对白一字不改
- 舞台指示完整保留
- 标注"剧本原词"前缀

### 5. v3.3 重构（v2.6 新增）
- **物理细节按时长分级**：≤5s ≤4 / 6s ≤6 / 7-8s ≤8 / 9-10s ≤8 / 11-15s ≤8
- **场景空间坐标段**：6 字段（拓扑/3人物坐标/位移/道具状态）替代 60+ 处文字锚点
- **黑名单词替换**：反作用力/抛物线/帧率切换/微距离 → 物理描述
- **M11 角色调用决策**：6 角色（AI 0.30/导演 0.20/摄影剪辑编剧 0.15/制片 0.05）加权投票
- **本镜自检段**：6 项 M13 自检清单（物理细节数/补充段/文字锚点/场景坐标/机位方向/黑名单词）

### 6. v3.4 治本·M14 资产档案去重（NEW）
- **核心问题**：人物资产档案段文字描述与参考图重复（11 镜共 3831 字冗余），AI 在两个信号源间困惑
- **核心原则**：参考图已承载信息则提示词不重复
- **实施方案**：档案段改为"1 行声明 + N 行参考图指针"
- **保留项**：合规/禁忌信息（图片无法承载）
- **删除项**：年龄/身高/服装/发型/表情/道具外观/场景细节
- **工具链升级**：validate_prompt.py v2.0 → v2.1（M14 第 7 项自检）+ v34_refactor_archive.py 批量重构脚本
- **实测效果**：档案段字数 -18%（3831 → 3141），复杂镜（镜007+008/镜009A/镜010+011）节省 -30% ~ -44%

## 扩展

### 添加新风格
1. 在 `styles/` 下创建 `STYLE_xxx.md`
2. 遵循统一 Schema
3. 在 SKILL.md 中追加索引

### 添加新知识
1. 在 `knowledge-base/` 下创建 `KB0X_xxx.md`
2. 在相关模块中引用

### 添加新平台
1. 在 `KB06_AI视频平台矩阵.md` 中追加
2. 调整提示词兼容性

### v3.3 重构（新剧种）
1. 完成 M9 4 维比对（剧本-TXT Gap 分析）
2. 完成 M10 约束机制（v1-v8 基础约束）
3. 完成 M11 角色调用（5+ 模拟决策）
4. 完成 M12 幻觉率与时长分级
5. 完成 M13 v3.3 重构（拆分明细 + 关键修复）
6. 生成 v3.3 txt 文件到 `01_分镜头/v3.3_重构版/`
7. 运行 validate_prompt.py v2.0 + continuity_check.py v2.0 双重验证
8. 参考 KB09 应用指南，沉淀到 KB09 知识库
